<a name="wcag-overview"></a>
## 📖 WCAG AA 기준 개요

### WCAG란?

**Web Content Accessibility Guidelines (WCAG)**는 웹 콘텐츠의 접근성을 보장하기 위한 국제 표준입니다.

**WCAG 2.1 AA**는 다음을 보장합니다:
- 시각 장애인도 콘텐츠를 이해할 수 있음
- 색맹 사용자도 정보를 구분할 수 있음
- 저시력 사용자도 텍스트를 읽을 수 있음
- 노인 사용자도 편안하게 사용할 수 있음

### 교육용 이미지 생성 시 중요한 기준

1. **색상 대비 (Contrast)**: 텍스트와 배경 간 명도 대비 4.5:1 이상
2. **색상 독립성**: 색상만으로 정보를 전달하지 않음
3. **텍스트 크기**: 본문 최소 16pt, 라벨 최소 14pt
4. **명확성**: 간단하고 이해하기 쉬운 언어 사용

<a name="color-contrast"></a>
## 🎨 색상 대비 (Color Contrast)

### WCAG AA 대비 기준

| 요소 | 최소 대비 | 권장 대비 | 예시 |
|------|----------|----------|------|
| **본문 텍스트** | 4.5:1 | 7:1 | 검은 글자 (0,0,0) + 흰 배경 (255,255,255) = 21:1 ✅ |
| **큰 텍스트** (18pt+) | 3:1 | 4.5:1 | 회색 글자 (118,118,118) + 흰 배경 = 3.5:1 ✅ |
| **라벨/아이콘** | 3:1 | 4.5:1 | 파란색 (0,0,255) + 흰 배경 = 8.6:1 ✅ |

### 프롬프트에 추가할 표현

#### 높은 대비 명시

**권장 표현**:
```
"높은 색상 대비 (4.5:1 이상), 명확한 명도 차이"
```

**구체적 예시**:
```
"검은색 텍스트와 흰색 배경, 또는 짙은 파란색과 밝은 배경으로
WCAG AA 기준 4.5:1 이상의 명도 대비를 보장"
```

#### 색맹 친화적 색상 조합

**권장 색상 조합**:
- 파란색 + 주황색 (보색 대비)
- 초록색 + 빨간색 (단, 명도 차이 크게)
- 파란색 + 노란색

**프롬프트 표현**:
```
"색맹 친화적 색상 조합 (파란색과 주황색 같은 보색 사용),
색상만으로 정보를 전달하지 않고 패턴과 라벨을 병행"
```

### 스타일별 대비 최적화

#### 핸드드로잉 스케치
```
"검은 마커와 다채로운 색연필로 명확한 대비,
흰색 또는 크림색 배경에 짙은 선과 밝은 채움으로 구분"
```

#### 미니멀 플랫
```
"그라데이션 없이 순수한 색상 블록 사용,
각 색상 간 충분한 명도 차이 보장 (최소 3:1)"
```

#### 기술 도면
```
"단색(검은색 또는 파란색) 선과 흰색/밝은 배경으로
최대 대비 (21:1) 보장"
```

#### 그라디언트 현대 기술
```
"어두운 배경에 밝은 네온 색상으로 강한 대비,
텍스트는 항상 밝은 색상 (흰색, 밝은 청색) 사용"
```

<a name="korean-rendering"></a>
## 🇰🇷 한글 렌더링 최적화

### 한글 폰트 최적화

#### 권장 서체

| 서체 타입 | 권장 폰트 | 용도 | 예시 |
|----------|----------|------|------|
| **고딕체** (산세리프) | 맑은 고딕, Noto Sans KR | 본문, 라벨 | 명확한 가독성 |
| **명조체** (세리프) | 나눔명조, Noto Serif KR | 제목 (제한적) | 전통적 느낌 |
| **손글씨** | 배달의민족 주아, 나눔손글씨 | 핸드드로잉 스타일 | 친근한 느낌 |

#### 프롬프트 표현

**권장**:
```
"명확한 한글 서체(고딕 계열), 충분한 폰트 크기,
자모 분리가 명확한 깔끔한 타이포그래피"
```

**구체적 예시**:
```
"맑은 고딕 또는 Noto Sans KR 같은 산세리프 한글 폰트,
최소 16pt 크기, 자음과 모음이 명확하게 구분되는 깔끔한 렌더링"
```

### 폰트 크기 기준

| 요소 | 최소 크기 | 권장 크기 | 비고 |
|------|----------|----------|------|
| **본문 텍스트** | 16pt | 18-20pt | 긴 설명 |
| **라벨** | 14pt | 16pt | 짧은 키워드 |
| **제목** | 24pt | 28-32pt | 섹션 제목 |
| **주석** | 12pt | 14pt | 부가 설명 |

#### 프롬프트 표현

```
"본문 텍스트 최소 16pt, 라벨 최소 14pt,
모든 한글 텍스트가 명확하게 읽히는 충분한 크기"
```

### 한글 렌더링 품질

#### 자모 분리

**권장 표현**:
```
"한글 자음과 모음이 명확하게 분리되어 표시,
겹받침과 복잡한 모음도 정확하게 렌더링"
```

#### 균형 잡힌 자간/행간

```
"적절한 자간과 행간으로 한글의 가독성 극대화,
글자 간 충분한 여백으로 밀집되지 않은 편안한 배치"
```

<a name="text-density"></a>
## 📝 텍스트 밀도와 가독성

### 텍스트 양 제한

#### 일행 최대 글자 수

| 매체 | 한글 기준 | 영어 기준 | 이유 |
|------|----------|----------|------|
| **슬라이드** | 최대 35자 | 최대 50자 | 멀리서 읽기 |
| **인쇄물** | 최대 45자 | 최대 65자 | 눈의 이동 최소화 |
| **라벨** | 최대 10자 | 최대 15자 | 즉시 인식 |

#### 프롬프트 표현

```
"일행 최대 35자 이내의 짧은 텍스트,
라벨은 간단명료하게 10자 이하로 제한"
```

### 정보 밀도

#### 과도한 텍스트 피하기

**비권장** (텍스트 과다):
```
"긴 문단 설명이 가득한 이미지, 여러 줄의 상세한 설명"
```

**권장** (적절한 밀도):
```
"핵심 키워드만 간단하게 라벨링, 시각적 요소가 주를 이루고
텍스트는 최소한으로 보조"
```

#### 프롬프트 표현

```
"과도한 텍스트를 피하고 시각적 요소 중심,
핵심 키워드만 짧게 라벨링"
```

<a name="icons-labels"></a>
## 🏷️ 아이콘과 라벨

### 색상 독립성

#### 색상만으로 구분하지 않기

**비권장** (색상만 사용):
```
"빨간색은 에러, 초록색은 성공을 나타냄 (라벨 없음)"
```

**권장** (색상 + 라벨 + 패턴):
```
"빨간색 박스에 '에러' 라벨과 X 아이콘,
초록색 박스에 '성공' 라벨과 체크 아이콘"
```

#### 프롬프트 표현

```
"색상만으로 정보를 전달하지 않고, 모든 색상 구분에
명확한 한글 라벨과 아이콘을 병행하여 색맹 사용자도 이해 가능"
```

### 아이콘 명확성

#### 직관적인 아이콘

**권장 아이콘**:
- ✅ 체크마크 = 성공, 완료
- ❌ X 표시 = 에러, 실패
- ⚠️ 경고 표시 = 주의
- ➡️ 화살표 = 방향, 흐름

#### 프롬프트 표현

```
"명확하고 직관적인 아이콘 (체크, X, 화살표 등),
각 아이콘에 한글 라벨이 항상 함께 표시"
```

### 범례 포함

#### 복잡한 다이어그램

**권장 표현**:
```
"이미지 하단 또는 측면에 색상과 기호의 의미를 설명하는
범례(Legend) 포함, 각 항목이 명확한 한글로 설명됨"
```

<a name="accessibility-checklist"></a>
## ✅ 접근성 검증 체크리스트

### 생성 전 검증

모든 프롬프트 생성 시 다음 항목을 확인하세요:

#### 1. 색상 대비 ✓

- [ ] 본문 텍스트: 4.5:1 이상 명도 대비
- [ ] 라벨: 최소 3:1 대비
- [ ] 색맹 친화적 색상 조합 (파란색 + 주황색 같은 보색)
- [ ] 색상만으로 정보 전달하지 않음 (라벨 병행)

**프롬프트에 추가**:
```
"높은 색상 대비 (WCAG AA 기준 4.5:1 이상),
색맹 친화적 색상 조합 (파란색과 주황색 같은 보색 사용)"
```

#### 2. 한글 렌더링 ✓

- [ ] 폰트 크기: 본문 최소 16pt, 라벨 최소 14pt
- [ ] 서체: 산세리프 (고딕 계열) 사용
- [ ] 자모 분리: 자음과 모음이 명확하게 구분
- [ ] 가독성: 충분한 자간과 행간

**프롬프트에 추가**:
```
"명확한 한글 서체(고딕 계열), 본문 최소 16pt, 라벨 최소 14pt,
자모 분리가 명확한 깔끔한 타이포그래피"
```

#### 3. 텍스트 밀도 ✓

- [ ] 일행 최대 35자 (한글 기준)
- [ ] 라벨은 간단명료 (10자 이하)
- [ ] 과도한 텍스트 피하기
- [ ] 시각적 요소 중심, 텍스트는 보조

**프롬프트에 추가**:
```
"일행 최대 35자 이내의 짧은 텍스트,
라벨은 간단명료하게 10자 이하"
```

#### 4. 아이콘과 라벨 ✓

- [ ] 색상만으로 구분하지 않음
- [ ] 모든 아이콘에 한글 라벨
- [ ] 직관적인 아이콘 사용
- [ ] 범례 또는 설명 포함 (복잡한 경우)

**프롬프트에 추가**:
```
"색상만으로 정보를 전달하지 않고, 모든 색상 구분에
명확한 한글 라벨과 아이콘을 병행"
```

### 생성 후 검증

생성된 이미지를 다음 기준으로 검증하세요:

#### 시각 장애 시뮬레이션

**도구**:
- **색맹 시뮬레이터**: Sim Daltonism (Mac), Color Oracle (Windows)
- **저시력 시뮬레이터**: NoCoffee Vision Simulator (Chrome Extension)
- **대비 검사**: WebAIM Contrast Checker

**검증 방법**:
1. 색맹 시뮬레이터로 이미지 확인
2. 모든 정보가 색상 없이도 구분되는지 확인
3. 대비 검사 도구로 텍스트-배경 대비 측정

#### 실제 사용자 테스트

**테스트 그룹**:
- 색맹 사용자
- 저시력 사용자
- 노인 사용자

**질문**:
- 모든 텍스트가 명확하게 읽히나요?
- 색상 구분이 어렵지 않나요?
- 정보를 이해하는 데 어려움이 있나요?

## 🎯 스타일별 접근성 가이드

### 핸드드로잉 스케치

**접근성 강점**:
- 친근하고 이해하기 쉬움
- 유기적인 화살표로 흐름 명확

**주의사항**:
- 손글씨 가독성 확보 (충분한 크기)
- 색연필 색상 대비 확인

**프롬프트 추가 표현**:
```
"손글씨도 명확하게 읽히는 크기와 대비,
색연필 색상은 배경과 충분한 대비 보장"
```

### 미니멀 플랫

**접근성 강점**:
- 높은 대비 (그라데이션 없음)
- 명확한 색상 블록 구분

**주의사항**:
- 색상만으로 구분하지 않도록 라벨 필수

**프롬프트 추가 표현**:
```
"각 색상 블록에 명확한 한글 라벨,
색상만으로 정보를 전달하지 않음"
```

### 기술 도면 블루프린트

**접근성 강점**:
- 최대 대비 (단색 선 + 밝은 배경)
- 정밀한 라벨링

**주의사항**:
- 너무 작은 글씨 피하기
- 복잡한 선이 겹치지 않도록

**프롬프트 추가 표현**:
```
"모든 주석과 라벨이 충분한 크기 (최소 14pt),
선이 겹치지 않고 명확하게 구분"
```

### 그라디언트 현대 기술

**접근성 강점**:
- 밝은 네온 글로우로 강조
- 어두운 배경과 강한 대비

**주의사항**:
- 그라디언트가 텍스트 가독성 방해하지 않도록
- 네온 효과가 과하지 않도록

**프롬프트 추가 표현**:
```
"텍스트는 항상 밝은 단색 (흰색 또는 밝은 청색),
그라디언트 배경이 텍스트 가독성을 방해하지 않도록 충분한 대비"
```
