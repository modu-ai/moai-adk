# 작성 완료 후 품질 체크리스트

**목적**: 절 작성 완료 후 필수 검증 항목

---

## ✅ 체크리스트 1: 인용 시스템

- [ ] 모든 통계와 사실에 인용 번호 추가됨 (예: "78% 개발자가 사용한다 (1)")
- [ ] 절 하단에 "인용문" 섹션 존재
- [ ] 인용 번호와 인용문 목록 일치 (본문의 (1)이 인용문 1번과 일치)
- [ ] 모든 인용문에 URL 포함
- [ ] 발행일 및 저자 명시

---

## ✅ 체크리스트 2: 내용 분량

- [ ] 절 전체 1,500 단어 이상
- [ ] 각 소절 500 단어 이상
- [ ] 최소 10개 문단
- [ ] 최소 3개 코드 예제 (기초, 응용, 고급)

---

## ✅ 체크리스트 3: 깊이 및 구체성

- [ ] 피상적 설명 없음 (2문장 미만 문단 금지)
- [ ] 모든 기술 용어에 설명 추가 (예: "터미널 (Terminal)")
- [ ] 구체적 예시 포함 (숫자, 파일명, 명령어)
- [ ] 비유 및 실생활 예시 활용

---

## ✅ 체크리스트 4: AI 표현 제거

- [ ] "물론", "당연히", "매우", "아주" 등 과장 표현 제거
- [ ] "혁신적", "획기적", "놀라운" 등 AI스러운 형용사 제거
- [ ] "~할 수 있습니다" → "~한다" (단정적 표현으로 변경)
- [ ] 추상적 표현 → 구체적 표현 (예: "많은 기능" → "4가지 핵심 기능")

---

## ✅ 체크리스트 5: 할루시네이션 검증

- [ ] 모든 버전 번호 공식 문서 확인 (예: "v1.0.58")
- [ ] 모든 통계 출처 확인 (Stack Overflow Survey 등)
- [ ] 모든 기능 설명 공식 문서 대조
- [ ] 추측성 표현 제거 ("~일 것이다", "~로 보인다" 삭제)

---

- [ ] 문장 길이 적절 (평균 20-30 단어)

### 시각적 요소

### 코드 예제

- [ ] 모든 코드 예제 실행 가능
- [ ] 코드 주석 추가 (설명)
- [ ] 코드 출처 표시 (공식 문서 또는 GitHub)
