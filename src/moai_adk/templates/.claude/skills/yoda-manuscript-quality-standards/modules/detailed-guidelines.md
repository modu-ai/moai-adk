## 1️⃣ 3축 템플릿 시스템 준수 검증 (신규)

**4단계 필수 조사**:

```
Step 1: Context7 MCP → 공식 API 문서 우선
  mcp__context7__resolve-library-id("Claude Code")
  mcp__context7__get-library-docs(libraryID, topic)

Step 2: WebSearch (공식 출처만) → Anthropic, Claude 공식 사이트
  WebSearch(query, allowed_domains=[
    "anthropic.com", "docs.anthropic.com", "claude.ai",
    "github.com/anthropics"
  ])

Step 3: 최신성 검증 (6개월 이내 우선)
  - 발행일 확인
  - 버전 정보 확인 (Claude Code v1.0+)

Step 4: 공식 출처 검증 (필수)
  - 공식 문서 우선
  - 불확실한 정보 제외
  - 개인 블로그/커뮤니티 인용 금지
```

**허용 도메인**:
- ✅ Anthropic 공식: anthropic.com, docs.anthropic.com
- ✅ Claude 공식: claude.ai, docs.claude.com
- ✅ GitHub 공식: github.com/anthropics
- ⚠️ 공식 통계만: stackoverflow.blog (Developer Survey)
- ❌ 금지: Medium, Dev.to, Reddit, 개인 블로그

📖 **상세**: `@reference/research-process.md`

### 2️⃣ 인용 시스템

**기본 규칙**:

```markdown
본문 작성 시:
Claude Code는 2024년 중반에 출시되었다 (1).
200K 토큰 컨텍스트 윈도우를 지원한다 (1).
78% 개발자가 AI 도구를 사용한다 (2).

절 하단 (반드시 추가):
## 인용문

1. Anthropic. "Claude Code Overview". *Claude Code Official Documentation*. 2025-11-20. [https://docs.anthropic.com/claude-code](https://docs.anthropic.com/claude-code)

2. Stack Overflow. "Developer Survey 2025". *Stack Overflow Official Blog*. 2025-05-15. [https://stackoverflow.blog/2025/survey](https://stackoverflow.blog/2025/survey)
```

**인용이 필요한 정보**:
- ✅ 통계 및 수치 ("78% 개발자가...")
- ✅ 공식 정의 및 용어
- ✅ 기술 사양 ("200K 토큰")
- ✅ 출시일 및 버전 정보
- ✅ 벤치마크 결과
- ❌ 일반 상식, 저자 의견, 비유

📖 **상세**: `@reference/citation-system.md`

### 3️⃣ 내용 분량 기준

**정량적 기준**:

| 분량 단위 | 최소 기준 | 권장 기준 | 최대 기준 |
|---------|---------|---------|---------|
| **한글 단어** | 1,500 단어 | 2,000 단어 | 2,500 단어 |
| **문단** | 10 문단 | 15 문단 | 20 문단 |
| **코드 예제** | 2개 | 3개 | 5개 |
| **인용** | 3개 | 5개 | 10개 |

**문단 구성**:
- 최소 3문장 이상
- 최대 7문장 이하
- 평균 80-120 단어
- 하나의 핵심 아이디어만 다룸

**문장 길이**:
- 평균: 20-30 단어
- 최대: 50 단어 (복잡한 기술 설명)
- 단문과 장문 적절히 혼합

**깊이 있는 설명 원칙**:
```
❌ 피상적 설명:
"Claude Code는 AI 코딩 도구다. 자연어로 요청하면 코드를 생성한다."
(2문장, 20단어 - 너무 짧음)

✅ 깊이 있는 설명:
"Claude Code는 Anthropic이 개발한 터미널 기반 AI 코딩 어시스턴트로,
2024년 중반에 정식 출시되었다 (1). 이 도구의 가장 큰 특징은
자연어로 요청을 받아 자율적으로 코드를 생성하고 실행하며,
오류가 발생하면 자동으로 디버깅까지 수행한다는 점이다 (1).

예를 들어, 'Python으로 사용자 로그인 기능을 구현해줘'라고 요청하면
Claude Code는 필요한 모든 파일을 생성하고, 데이터베이스 연결 코드를 작성하며,
보안 검증 로직을 추가하고, 테스트 코드까지 자동으로 작성하여 실행한다 (1)."
(5문단, 200+ 단어, 3개 인용 - 깊이 있고 구체적)
```

📖 **상세**: `@reference/content-quantity.md`

### 4️⃣ 품질 검사 기준 (5개 체크리스트)

**작성 후 필수 검사**:

#### ✅ 체크리스트 1: 인용 시스템
- [ ] 모든 통계와 사실에 인용 번호 추가됨
- [ ] 절 하단에 "인용문" 섹션 존재
- [ ] 인용 번호와 인용문 목록 일치
- [ ] 모든 인용문에 URL 포함
- [ ] 발행일 및 저자 명시

#### ✅ 체크리스트 2: 내용 분량
- [ ] 절 전체 1,500 단어 이상
- [ ] 각 소절 500 단어 이상
- [ ] 최소 10개 문단
- [ ] 최소 3개 코드 예제

#### ✅ 체크리스트 3: 깊이 및 구체성
- [ ] 피상적 설명 없음 (2문장 미만 문단 금지)
- [ ] 모든 기술 용어에 설명 추가
- [ ] 구체적 예시 포함 (숫자, 파일명, 명령어)
- [ ] 비유 및 실생활 예시 활용

#### ✅ 체크리스트 4: AI 표현 제거
- [ ] "물론", "당연히", "매우", "아주" 등 과장 표현 제거
- [ ] "혁신적", "획기적", "놀라운" 등 AI스러운 형용사 제거
- [ ] "~할 수 있습니다" → "~한다" (단정적 표현)
- [ ] 추상적 표현 → 구체적 표현

**AI 표현 제거 예시**:
```
❌ "매우 빠른 성능" → ✅ "평균 2-4초 응답 속도 (1)"
❌ "획기적인 도구" → ✅ "2024년 중반 출시된 터미널 기반 도구 (1)"
❌ "놀라운 결과" → ✅ "78% 개발자가 사용하는 도구 (2)"
❌ "많은 기능" → ✅ "4가지 핵심 기능 (코드 이해, 생성, 수정, Git 통합)"
```

#### ✅ 체크리스트 5: 할루시네이션 검증
- [ ] 모든 버전 번호 공식 문서 확인
- [ ] 모든 통계 출처 확인
- [ ] 모든 기능 설명 공식 문서 대조
- [ ] 추측성 표현 제거 ("~일 것이다", "~로 보인다")

**할루시네이션 방지**:
```
❌ "Claude Code는 모든 프로그래밍 언어를 지원한다"
✅ "Claude Code는 Python, JavaScript, TypeScript, Java, Go 등 20개 이상의 언어를 지원한다 (1)"

❌ "곧 새로운 기능이 출시될 것이다"
✅ "2025년 11월 현재, Claude Code v1.0.58이 최신 버전이다 (1)"
```

📖 **상세**: `@reference/quality-checklists.md`

### 5️⃣ 문체 및 스타일

**반말 (한다체) 규칙**:

```
종결어미:
- 평서문: ~이다, ~한다, ~된다
- 의문문: ~인가?, ~하는가?
- 명령문: ~하자, ~해보자
- 감탄문: ~구나!, ~네!

예시:
❌ "Claude Code를 설치해야 합니다." (존댓말)
✅ "Claude Code를 설치해야 한다." (반말)

❌ "이제 실행해볼까요?" (존댓말)
✅ "이제 실행해보자." (반말)
```

**친근하면서 전문적인 톤**:
- 격려하는 표현: "걱정하지 말자", "천천히 따라오면 된다"
- 전문성 유지: 기술 용어 정확히 사용, 영어 용어 병기
- 코드 예제: 실행 가능한 수준

**금지 표현** (절대 사용 금지):
- ❌ "매우", "아주", "정말"
- ❌ "물론", "당연히"
- ❌ "혁신적", "획기적", "놀라운"
- ❌ "~일 것입니다", "~로 보입니다" (추측)
- ❌ "~할 수 있습니다" (존댓말)

📖 **상세**: `@reference/style-guide.md`

### 6️⃣ 이미지 및 다이어그램

**절(Section)당 개수**:
- 최소: 1개
- 권장: 2-3개
- 최대: 4개

**핸드 드로잉 표준 템플릿** (필수 사용):

```
핸드 드로잉 삽화 스타일의 '[제목]' 인포그래픽입니다.
종이 질감 배경에 색연필로 그린 듯한 따뜻한 색감과 손으로 그린 선,
유기적인 화살표가 특징입니다.
[구체적 설명]

주요 키워드 및 요소:
- 스타일: 핸드 드로잉, 스케치 노트, 만화, 따뜻한 색연필 질감
- 배경: 종이 질감 (크림색 또는 베이지)
- 텍스트: 손글씨 스타일
- 요소: [구체적 요소]
- 전반적인 느낌: 친근하고 따뜻하며 정보 전달력이 높은 인포그래픽
```

**Mermaid 다이어그램 활용**:
- 간단한 플로우: 3-5단계 플로우차트
- 시퀀스 다이어그램
- 상태 다이어그램

**이미지 vs Mermaid 선택**:
- Mermaid: 6개 미만 컴포넌트, 간단한 관계
- 이미지: 6개 이상 컴포넌트, Before-After 비교, 터미널 화면

📖 **상세**: `@reference/visual-elements.md`
