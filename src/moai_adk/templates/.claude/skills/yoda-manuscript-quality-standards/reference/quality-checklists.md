# 품질 검사 기준

**목적**: 일관된 품질 표준 준수 및 검증

---

## 작성 후 필수 검사 항목

### ✅ 체크리스트 1: 인용 시스템

- [ ] 모든 통계와 사실에 인용 번호 추가됨
- [ ] 절 하단에 "인용문" 섹션 존재
- [ ] 인용 번호와 인용문 목록 일치
- [ ] 모든 인용문에 URL 포함
- [ ] 발행일 및 저자 명시

### ✅ 체크리스트 2: 내용 분량

- [ ] 절 전체 1,500 단어 이상
- [ ] 각 소절 500 단어 이상
- [ ] 최소 10개 문단
- [ ] 최소 3개 코드 예제

### ✅ 체크리스트 3: 깊이 및 구체성

- [ ] 피상적 설명 없음 (2문장 미만 문단 금지)
- [ ] 모든 기술 용어에 설명 추가
- [ ] 구체적 예시 포함 (숫자, 파일명, 명령어)
- [ ] 비유 및 실생활 예시 활용

### ✅ 체크리스트 4: AI 표현 제거

- [ ] "물론", "당연히", "매우", "아주" 등 과장 표현 제거
- [ ] "혁신적", "획기적", "놀라운" 등 AI스러운 형용사 제거
- [ ] "~할 수 있습니다" → "~한다" (단정적 표현)
- [ ] 추상적 표현 → 구체적 표현

### ✅ 체크리스트 5: 할루시네이션 검증

- [ ] 모든 버전 번호 공식 문서 확인
- [ ] 모든 통계 출처 확인
- [ ] 모든 기능 설명 공식 문서 대조
- [ ] 추측성 표현 제거 ("~일 것이다", "~로 보인다")

## AI 표현 제거 가이드

### 과장 표현 (❌ → ✅)

- ❌ "매우 빠른 성능" → ✅ "평균 2-4초 응답 속도 (1)"
- ❌ "획기적인 도구" → ✅ "2024년 중반 출시된 터미널 기반 도구 (1)"
- ❌ "놀라운 결과" → ✅ "78% 개발자가 사용하는 도구 (2)"

### 추상적 표현 (❌ → ✅)

- ❌ "많은 기능" → ✅ "4가지 핵심 기능 (코드 이해, 생성, 수정, Git 통합)"
- ❌ "다양한 프로젝트" → ✅ "10개 이상 파일의 멀티파일 프로젝트"
- ❌ "효율적인 작업" → ✅ "평균 30분 만에 로그인 기능 완성 (사례 연구 기준)"

### AI스러운 형용사 (❌ → ✅)

- ❌ "혁신적인" → ✅ "에이전틱 시스템 기반의"
- ❌ "강력한" → ✅ "200K 토큰 컨텍스트 윈도우를 제공하는"
- ❌ "완벽한" → ✅ "78% 정확도의 (SWE-Bench 기준)"

## 할루시네이션 방지 규칙

1. 공식 문서에 없는 기능 언급 금지
2. 검증되지 않은 통계 사용 금지
3. 추측성 미래 전망 금지
