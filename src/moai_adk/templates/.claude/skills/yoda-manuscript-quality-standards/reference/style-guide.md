# 문체 및 스타일

**목적**: 일관되고 친근한 문체 유지

---

## 1. 반말 (한다체) 규칙

### 종결어미

- 평서문: ~이다, ~한다, ~된다
- 의문문: ~인가?, ~하는가?
- 명령문: ~하자, ~해보자
- 감탄문: ~구나!, ~네!

### 예시

```
❌ "Claude Code를 설치해야 합니다." (존댓말)
✅ "Claude Code를 설치해야 한다." (반말)

❌ "이제 실행해볼까요?" (존댓말)
✅ "이제 실행해보자." (반말)
```

---

## 2. 친근하면서 전문적인 톤

### 격려하는 표현

- "걱정하지 말자"
- "천천히 따라오면 된다"
- "어렵지 않다"

### 전문성 유지

- 기술 용어는 정확히 사용
- 영어 용어는 한글 용어 뒤에 병기 (예: "터미널 (Terminal)")
- 코드 예제는 실행 가능한 수준

---

## 3. 금지 표현

### ❌ 절대 사용 금지

- "매우", "아주", "정말"
- "물론", "당연히"
- "혁신적", "획기적", "놀라운"
- "~일 것입니다", "~로 보입니다" (추측)
- "~할 수 있습니다" (존댓말)

### 올바른 대체 표현

| 금지 표현 | 올바른 표현 |
|---------|-----------|
| "매우 중요하다" | "2024년 78% 개발자가 사용한다 (1)" |
| "혁신적인 도구" | "2024년 중반 출시된 도구 (1)" |
| "완벽하게 작동한다" | "78% 정확도로 작동한다 (SWE-Bench 기준) (2)" |
| "많은 개발자가 사용한다" | "전 세계 50만 개발자가 사용한다 (3)" |
