# 작성 워크플로우 (4단계)

**목적**: 체계적이고 효율적인 원고 작성 프로세스

---

## Phase 1: 자료 조사 (30-40%)

### 단계별 실행

1. **Context7 MCP로 공식 문서 조사**
   ```python
   mcp__context7__resolve-library-id("Claude Code")
   mcp__context7__get-library-docs(libraryID, topic="핵심 기능")
   ```

2. **WebSearch로 최신 자료 조사 (공식 출처만)**
   ```python
   WebSearch(
       query="Claude Code 최신 기능",
       allowed_domains=[
           "anthropic.com",
           "docs.anthropic.com",
           "github.com/anthropics"
       ]
   )
   ```

3. **조사 결과 문서화**
   - 출처, 제목, URL, 발행일 기록
   - 공식 출처 여부 확인
   - 인용 가능 정보 추출

4. **인용 목록 초안 작성**
   - 인용 번호 순서대로 정리
   - 형식: 번호. 저자. "제목". *출처*. 날짜. [URL]

---

## Phase 2: 초안 작성 (40-50%)

### 단계별 실행

1. **개요 작성 (주요 섹션 구조)**
   - 소절(Subsection) 제목 정리
   - 각 소절의 핵심 메시지 정의
   - 논리적 흐름 확인

2. **각 소절 작성 (최소 500 단어)**
   - 정의 + 비유 + 예시
   - 구체적인 수치와 사례
   - 인용 번호 삽입

3. **코드 예제 추가 (기초→응용→고급)**
   - 기초: 5-10줄 (기본 개념)
   - 응용: 10-15줄 (실제 사용)
   - 고급: 15-25줄 (복잡한 시나리오)

4. **인용 번호 삽입 (본문에 (1), (2) 추가)**
   - 통계, 사실, 기술 사양에 반드시 추가
   - 동일 출처는 같은 번호 사용

---

## Phase 3: 품질 검사 (10-20%)

### 단계별 실행

1. **자동 체크리스트 검증 (5개)**
   - 인용 시스템
   - 내용 분량
   - 깊이 및 구체성
   - AI 표현 제거
   - 할루시네이션 검증

2. **AI 표현 제거**
   - "매우", "아주" → 구체적 수치
   - "혁신적" → 출시일 + 특징
   - "~할 수 있습니다" → "~한다"

3. **할루시네이션 검증**
   - 모든 버전 번호 확인
   - 모든 통계 출처 확인
   - 추측성 표현 제거

4. **분량 확인**
   - 1,500 단어 이상
   - 10 문단 이상
   - 3개 코드 예제 이상

5. **인용문 섹션 완성**
   - 절 하단에 "인용문" 섹션 추가
   - 모든 인용 출처 나열
   - URL 및 발행일 포함

---

## Phase 4: 최종 검토 (5-10%)

### 단계별 실행

1. **전체 흐름 확인**
   - 논리적 연결성
   - 섹션 간 일관성
   - 중복 내용 제거

2. **문체 일관성 점검**
   - 반말 (한다체) 일관성
   - 기술 용어 일관성
   - 문장 길이 적절성

3. **이미지 프롬프트 추가**
   - 핸드 드로잉 템플릿 사용
   - 절당 2-3개 권장

4. **최종 저장**
   - 파일명 규칙 준수
   - 메타데이터 업데이트

---
