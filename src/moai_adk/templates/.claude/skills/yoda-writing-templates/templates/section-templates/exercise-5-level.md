# 섹션 템플릿: exercise-5-level (연습 문제 - 5단계)

## 📋 용도

본문 끝, 이해도 확인 및 응용력 검증

**구조**: Level 1 (이해) → Level 2 (기본) → Level 3 (중급) → Level 4 (응용) → Level 5 (창의)

**특징**: 점진적 난이도 증가

---

## 🏗️ 5가지 구성 요소

### 1. 문서 구조

```
## 연습 문제 📝

### Level 1: 이해 확인 (2분)
1. 문제 1
2. 문제 2

### Level 2: 기본 실습 (5분)
3. 문제 3
4. 문제 4

### Level 3: 응용 (10분)
5. 문제 5

### Level 4: 심화 (15분)
6. 문제 6

### Level 5: 창의 (20분)
7. 문제 7

(총 7-10개 문제)
```

### 2. 문체

**어조**: 명령형, 격려하는

**종결어미**: "-세요", "-해보세요"

**능동태**: 100%

**문장 길이**: 평균 15단어

### 3. 내용 전개

**난이도 증가**:
- Level 1: 이해 (개념 확인)
- Level 2: 기본 (간단한 실습)
- Level 3: 응용 (복합 문제)
- Level 4: 심화 (실전 시나리오)
- Level 5: 창의 (자유 프로젝트)

**시간 배분**:
- Level 1: 2분
- Level 2: 5분
- Level 3: 10분
- Level 4: 15분
- Level 5: 20분

### 4. 조건

**문제 개수**:
- Level 1: 2개
- Level 2: 2개
- Level 3: 1-2개
- Level 4: 1개
- Level 5: 1개

**총 7-10개 문제**

**필수 요소**:
- 각 Level마다 시간 명시
- 명확한 문제 설명
- 구체적 목표

### 5. 형식

```markdown
## 연습 문제 📝

### Level 1: 이해 확인 (2분)
1. [개념 A]와 [개념 B]의 차이점을 설명해보세요.
2. [용어 X]의 정의를 한 문장으로 작성하세요.

### Level 2: 기본 실습 (5분)
3. [기능 Y]를 구현하는 코드를 작성해보세요.
4. [도구 Z]를 설치하고 버전을 확인해보세요.

### Level 3: 응용 (10분)
5. [시나리오 W]를 해결하는 프로그램을 작성하세요.

### Level 4: 심화 (15분)
6. [복잡한 요구사항]을 만족하는 솔루션을 설계하고 구현하세요.

### Level 5: 창의 (20분)
7. 자신만의 [프로젝트 아이디어]를 생각하고 완성해보세요.
```

## 📝 작성 프롬프트 템플릿

```
연습 문제 (5단계) 섹션을 작성하세요.

**구조**:
- Level 1: 이해 확인 (2분, 2문제)
- Level 2: 기본 실습 (5분, 2문제)
- Level 3: 응용 (10분, 1-2문제)
- Level 4: 심화 (15분, 1문제)
- Level 5: 창의 (20분, 1문제)

**총 문제 개수**: 7-10개

**조건**:
- 각 Level마다 시간 명시
- 명령형 ("-세요")
- 구체적 목표

챕터 주제: [주제]
학습 목표: [목표 3-5개]
```
