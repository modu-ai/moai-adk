# PART 4 템플릿: project-walkthrough (프로젝트 워크스루)

## 📋 용도

**PART 4 (Ch 15-20)** 실전 프로젝트 구축 시 사용

**대상 독자**: 중급-고급 (2-5년 경험)

**주요 특징**:
- 실전적이고 동료와 함께하는 톤
- 목표 → 준비 → Step 1-3 → 완성
- 각 Step마다 코드 + 실행 결과
- 체크포인트와 트러블슈팅

---

## 🏗️ 5가지 구성 요소

### 1. 문서 구조

```
총 구조:
├─ 프로젝트 목표 (1 섹션, 1-2 단락, 각 3-4 문장)
├─ 준비 (1 섹션, 2 단락, 각 3-4 문장)
├─ Step 1 (2 섹션, 각 2-3 단락, 각 4-5 문장)
├─ Step 2 (2 섹션, 각 2-3 단락, 각 4-5 문장)
├─ Step 3 (2 섹션, 각 2-3 단락, 각 4-5 문장)
├─ 완성 확인 (1 섹션, 1-2 단락, 각 3-4 문장)
└─ 회고 (1 섹션, 1 단락, 3-4 문장)

총 섹션: 10-11개
총 단락: 15-20개
평균 문장/단락: 4개
```

### 2. 문체

**어조**: 실전적이고 동료와 함께
- "이제 만들어봅시다"
- "함께 완성해볼까요?"
- "잘 작동하네요!"

**종결어미**:
- "-요" (60%)
- "-다" (40%)

**능동태 비율**: 90% 이상

**문장 길이**: 평균 20단어

**전문 용어**: 실무 용어 (한국어 우선)
- "컴포넌트"
- "API 엔드포인트"

### 3. 내용 전개

**배치 순서**:
1. 목표 ("무엇을 만들까요?")
2. 준비 (설치, 설정)
3. Step 1 (기본 구조)
4. Step 2 (기능 추가)
5. Step 3 (완성 및 최적화)
6. 완성 확인
7. 회고

**깊이**: Level 2 (중급) 50% + Level 3 (고급) 50%

**예제 복잡도**:
- Step 1: 10줄
- Step 2: 20줄
- Step 3: 35줄

**연결성**: 이전 프로젝트 40% 참조

### 4. 조건

**글자 수**: 2500-3500자

**코드 예제**: 7-10개 (각 Step마다 2-3개, 각 15-35줄)

**대상 독자**: 중급-고급 (2-5년)

**필수 요소**:
- 목표 1개
- 준비 1개
- Step 1-3 (각 2개 섹션)
- 완성 확인 1개
- 회고 1개

**선택 요소**:
- 트러블슈팅 2-3개
- 최적화 팁 1-2개

### 5. 형식

```markdown
# Chapter N: 프로젝트 제목

## 프로젝트 목표 🎯

### 무엇을 만들까요?

- 항목 1
- 항목 2

## Step 1: 기본 구조 만들기

### 1-1: 디렉토리 생성
코드:
```bash
# 명령어 (5-10줄)
```

### 1-2: 기본 파일 작성
코드:
```typescript
// 실전 코드 (20-30줄, 상세 주석)
```

✅ 체크포인트: Step 1 완료

## Step 2: 기능 추가

## 완성 확인 ✅

### 실행 결과

## 회고: 배운 점 📝
```
