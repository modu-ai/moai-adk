# PART 3 템플릿: methodology-guide (방법론 가이드)

## 📋 용도

**PART 3 (Ch 10-14)** 방법론 및 워크플로우 설명 시 사용

**대상 독자**: 중급-고급 (1-5년 경험)

**주요 특징**:
- 체계적이고 설득력 있는 어조
- 배경 → 철학 → 워크플로우 구조
- 추상적 개념을 구체적으로 설명
- 비유와 사례 활용

---

## 🏗️ 5가지 구성 요소

### 1. 문서 구조

```
총 구조:
├─ 배경 (1-2 섹션, 각 2 단락, 각 4-5 문장)
├─ 핵심 개념 (3 섹션, 각 2-3 단락, 각 4-5 문장)
├─ 워크플로우 (2 섹션, 각 3 단락, 각 3-4 문장)
├─ 실습 (1 섹션, 2-3 단락, 각 4-5 문장)
└─ 회고 (1 섹션, 1-2 단락, 각 3-4 문장)

총 섹션: 8-9개
총 단락: 13-17개
평균 문장/단락: 4개
```

### 2. 문체

**어조**: 체계적이고 설득력 있게
- "이 방법론은..."
- "따라서 우리는..."
- "왜 중요한가?"

**종결어미**:
- "-다" (80%)
- "-니다" (20%)

**능동태 비율**: 80% 이상

**문장 길이**: 평균 24단어

**전문 용어**: 방법론 용어 (영어/한국어 병기)
- "SPEC-First TDD"
- "RED-GREEN-REFACTOR"

### 3. 내용 전개

**배치 순서**:
1. 배경 ("왜 이 방법론인가?")
2. 철학 (핵심 가치)
3. 핵심 개념 3가지
4. 워크플로우 (단계별 진행)
5. 실습 (적용 방법)
6. 회고 (배운 점)

**깊이**: Level 2 (중급) 50% + Level 3 (고급) 50%

**예제**: 워크플로우 예시 (15-25줄)

**연결성**: 이전 장 35% 참조

### 4. 조건

**글자 수**: 2200-3000자

**코드 예제**: 3-5개 (워크플로우 예시, 각 15-25줄)

**대상 독자**: 중급-고급 (1-5년)

**필수 요소**:
- 배경 1개
- 핵심 개념 3개
- 워크플로우 2개
- 실습 1개
- 회고 1개

**선택 요소**:
- 비교 분석 1개
- 사례 연구 1개

### 5. 형식

```markdown
# Chapter N: 방법론 제목

## N-1: 배경과 철학

### 왜 이 방법론인가? 🤔

### 핵심 개념 3가지

#### 1. 개념 A
**정의**: ...
**예시**: ...

### 워크플로우 🔄

1. 단계 1
2. 단계 2
3. 단계 3

코드 (워크플로우 예시):
```python
# 실전 패턴 (15-25줄)
```

### 실습: 적용해보기 💡

### 회고: 배운 점 📝
```
