# PART 2 템플릿: technical-deep-dive (기술 심화)

## 📋 용도

**PART 2 (Ch 6-9)** 고급 기능 및 원리 설명 시 사용

**대상 독자**: 중급 (1-3년 경험)

**주요 특징**:
- 전문적이고 정확한 어조
- 깊이 있는 원리 설명
- 상세한 코드 예제 (10-20줄)
- 심화 섹션 포함

---

## 🏗️ 5가지 구성 요소

### 1. 문서 구조

```
총 구조:
├─ 도입 (1 섹션, 1 단락, 4-5 문장)
├─ 전환 (1 섹션, 1-2 단락, 각 3-4 문장)
├─ 본문 섹션 1: 개념 (3 단락, 각 5-6 문장)
├─ 본문 섹션 2: 원리 (3 단락, 각 5-6 문장)
├─ 본문 섹션 3: 응용 (2-3 단락, 각 4-5 문장)
├─ 본문 섹션 4: 최적화 (2 단락, 각 4-5 문장)
├─ 심화 섹션 (선택, 2 단락, 각 3-4 문장)
└─ 정리 (1 섹션, 1-2 단락, 각 3-4 문장)

총 섹션: 7-8개
총 단락: 15-18개
평균 문장/단락: 4-5개
계층 깊이: H1 → H2 → H3 (필요 시 H4)
```

### 2. 문체

**어조**: 전문적이고 정확한
- "원리를 이해해야 합니다"
- "주의해야 할 점은..."
- "다음과 같이 동작합니다"

**종결어미**:
- "-다" (70%) - "작동한다", "필요하다"
- "-습니다" (30%) - "작동합니다", "필요합니다"

**능동태 비율**: 85% 이상

**문장 길이**: 평균 22단어

**전문 용어**: 영어 우선 (한국어 설명)
- "Async/Await (비동기 처리)"
- "Event Loop (이벤트 루프)"

### 3. 내용 전개

**배치 순서**:
1. 정의 ("X란 무엇인가?")
2. 원리 설명 ("어떻게 동작하는가?")
3. 코드 예제 (구체적 구현)
4. 응용 (실전 활용)
5. 최적화 (성능 개선)
6. 심화 (선택, 고급 주제)

**깊이 조절**:
- Level 2 (중급) 60%
- Level 3 (고급) 40%

**예제 복잡도**: 점진적 증가
- 기본 예제: 10줄
- 중급 예제: 15줄
- 고급 예제: 25줄

**연결성**: 이전 장 30% 참조

### 4. 조건

**글자 수**: 2000-2800자

**코드 예제**: 5-8개
- 각 10-20줄
- 상세한 주석 포함
- 실행 결과 포함

**대상 독자**: 중급 (1-3년)

**필수 요소**:
- 학습 목표 5개 (상세)
- 원리 설명 2개
- 심화 섹션 1개
- 요약

**선택 요소**:
- 성능 최적화 1개
- 보안 고려사항 1개

### 5. 형식

```markdown
# Chapter N: 제목

## N-1: 개념 이해

### 학습 목표 🎯
- 목표 1-5 (상세)

### 핵심 개념
**정의**: ...

> **원리**: ... (인용문으로 핵심 강조)

코드:
```typescript
// 중급 예제 (10-15줄, 상세 주석)
async function example() {
  // 설명
}
```

### 심화: 최적화 기법 ⚡

### 요약 📌
```

**아이콘**:
- 🎯 학습 목표
- ⚡ 성능/최적화
- 🔒 보안
