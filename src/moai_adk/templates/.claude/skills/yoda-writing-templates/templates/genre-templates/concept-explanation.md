# 장르 템플릿: concept-explanation (개념 설명형)

## 📋 용도

기초 개념 설명, 이해 중심

**구조**: 정의 → 왜 중요한가? → 원리 설명 → 실제 예시

**특징**:
- 비유와 다이어그램
- 단계별 설명
- 구체적 예시

---

## 🏗️ 5가지 구성 요소

### 1. 문서 구조

```
├─ 정의 (1 단락, 2-3 문장)
├─ 중요성 (1 단락, 3-4 문장)
├─ 원리 (2 단락, 각 4-5 문장)
└─ 예시 (1 단락, 3-4 문장)

총 단락: 5개
총 문장: 15-20개
```

### 2. 문체

**어조**: 설명하고 이해시키는

**종결어미**: "-다", "-습니다"

**능동태**: 80% 이상

**문장 길이**: 평균 20단어

### 3. 내용 전개

**배치 순서**:
1. 정의 ("X란 무엇인가?")
2. 중요성 ("왜 중요한가?")
3. 원리 ("어떻게 동작하는가?")
4. 예시 ("실제로는...")

### 4. 조건

**글자 수**: 600-1000자

**코드 예제**: 1-2개 (각 10-15줄)

**필수 요소**:
- 정의 1개
- 비유 1개
- 원리 설명
- 실제 예시 1개

### 5. 형식

```markdown
## N-1: 개념 이해

### X란 무엇인가?

**정의**: ...

> 비유: ... (인용문)

### 왜 중요한가?

### 동작 원리

코드:
```python
# 예시 (10-15줄)
```

### 실제 예시
```
