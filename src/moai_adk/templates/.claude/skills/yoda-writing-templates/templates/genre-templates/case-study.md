# 장르 템플릿: case-study (사례 분석형)

## 📋 용도

방법론 설명, 회고

**구조**: 배경 → 문제 상황 → 해결 방법 → 결과 → 교훈 도출

**특징**:
- 구체적인 사례
- 원인 분석
- 일반화된 교훈

---

## 🏗️ 5가지 구성 요소

### 1. 문서 구조

```
├─ 배경 (1 단락, 3-4 문장)
├─ 문제 상황 (2 단락, 각 3-4 문장)
├─ 해결 방법 (2 단락, 각 4-5 문장)
├─ 결과 (1 단락, 3-4 문장)
└─ 교훈 (1 단락, 3-4 문장)

총 단락: 7개
총 문장: 23-29개
```

### 2. 문체

**어조**: 분석적

**종결어미**: "-다", "-습니다"

**능동태**: 80% 이상

**문장 길이**: 평균 23단어

### 3. 내용 전개

**배치 순서**:
1. 배경 (상황 설정)
2. 문제 (무엇이 잘못되었나?)
3. 해결 (어떻게 해결했나?)
4. 결과 (무엇이 개선되었나?)
5. 교훈 (무엇을 배웠나?)

### 4. 조건

**글자 수**: 800-1200자

**코드 예제**: 1-2개 (해결 방법)

**필수 요소**:
- 구체적 사례 1개
- 원인 분석
- 교훈 3개

### 5. 형식

```markdown
## N-1: 사례 연구

### 배경

### 문제 상황 🚨

### 해결 방법 💡

코드:
```python
# 개선 후 (15-20줄)
```

### 결과 📊

Before vs After:
| 항목 | Before | After |
|------|--------|-------|
| A | X | Y |

### 교훈 📚

1. 교훈 1
2. 교훈 2
3. 교훈 3
```
