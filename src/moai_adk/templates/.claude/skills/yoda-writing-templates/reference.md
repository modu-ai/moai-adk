# yoda-writing-templates: 상세 레퍼런스

## 📚 개요

이 문서는 yoda-writing-templates Skill의 모든 템플릿이 갖추어야 할 **5가지 구성 요소**를 상세하게 설명합니다.

---

## 🏗️ 템플릿의 5가지 구성 요소

모든 템플릿은 다음 5가지 요소를 명확히 정의해야 합니다:

### 1. 문서 구조 (Document Structure)

**정의**: 챕터/섹션/단락/문장의 배치와 계층 구조

**세부 항목**:
- **섹션 수**: 몇 개의 주요 섹션으로 구성되는가?
- **섹션당 단락 수**: 각 섹션에 몇 개의 단락이 포함되는가?
- **단락당 문장 수**: 한 단락에 몇 개의 문장이 적절한가?
- **계층 깊이**: 제목의 계층(H1, H2, H3)이 몇 단계까지 내려가는가?

**예시**:
```
PART 1 basic-tutorial 구조:
├─ 도입 (1 섹션, 1-2 단락, 각 3-4 문장)
├─ 전환 (1 섹션, 1-2 단락, 각 2-3 문장)
├─ 본문 (3 섹션, 각 2-3 단락, 각 4-5 문장)
└─ 요약 (1 섹션, 1 단락, 3-4 문장)

총 섹션: 6개
총 단락: 11-14개
평균 문장/단락: 3-4개
```

---

### 2. 문체 (Writing Style)

**정의**: 어조, 종결어미, 능동태/수동태 비율, 문장 구조의 일관성

**세부 항목**:
- **어조 (Tone)**: 친근 / 전문 / 체계적 / 실전 중 선택
- **종결어미**: "-습니다" (격식) vs "-요" (친근) vs "-다" (간결)
- **능동태 비율**: 80% 이상 (읽기 쉬움) vs 50-70% (전문적)
- **문장 길이**: 평균 15-20단어 (짧음) vs 20-25단어 (중간) vs 25-30단어 (길음)
- **전문 용어**: 영어/한국어 병기 vs 한국어만 vs 영어만

**예시**:
```
PART 1 (친근한 어조):
- 어조: 격려하고 친근하게 ("함께 시작해볼까요?")
- 종결어미: "-요" (85%), "-습니다" (15%)
- 능동태 비율: 90% ("우리는 설치합니다" > "설치됩니다")
- 문장 길이: 평균 18단어
- 용어: 한국어 (영어 병기) - "프롬프트 (Prompt)"

PART 2 (전문적 어조):
- 어조: 정확하고 전문적 ("다음 원리를 이해해야 합니다")
- 종결어미: "-다" (70%), "-습니다" (30%)
- 능동태 비율: 85%
- 문장 길이: 평균 22단어
- 용어: 영어 우선 (한국어 설명) - "Async/Await (비동기 처리)"
```

---

### 3. 내용 전개 (Content Flow)

**정의**: 정보의 배치 순서, 학습 곡선 조절, 예제의 점진적 복잡도 증가

**세부 항목**:
- **정보 배치 순서**: 정의 → 원리 → 예제 vs 문제 → 솔루션 → 원리
- **깊이 조절**: 기초만 (Level 1) vs 기초+중급 (Level 2) vs 전문가까지 (Level 3)
- **예제 복잡도**: 5줄 → 10줄 → 20줄 vs 10줄 → 15줄 → 25줄
- **연결성**: 이전 장 참조 (20%) vs 독립적 (80%)

**예시**:
```
PART 1 (기초 중심):
정보 배치: 문제 제기 → 솔루션 제시 → 기초 개념 → 간단한 예제
깊이: Level 1 (기초만, 70%) + Level 2 (응용, 30%)
예제: 5줄 → 10줄 → 15줄 (점진적 증가)
연결성: 이전 장 20% 참조 ("앞서 배운...")

PART 4 (프로젝트):
정보 배치: 목표 → 준비 → Step 1 → Step 2 → Step 3 → 완성
깊이: Level 2 (중급, 60%) + Level 3 (고급, 40%)
예제: 10줄 → 20줄 → 35줄 (큰 증가폭)
연결성: 이전 장 40% 참조 ("지난 프로젝트에서...")
```

---

### 4. 조건 (Constraints)

**정의**: 글자 수, 코드 줄 수, 대상 독자, 복잡도, 시간 제약

**세부 항목**:
- **글자 수 범위**: 최소-최대 (예: 1500-2000자)
- **코드 예제 개수**: 몇 개의 코드 블록이 필요한가?
- **코드 줄 수**: 각 코드 블록의 최소/최대 줄 수
- **대상 독자**: 초급 (1년 미만) / 중급 (1-3년) / 고급 (3년 이상)
- **필수 포함 요소**: 학습 목표, 요약, 연습 문제 등
- **선택 요소**: 심화 내용, 사례, 트러블슈팅 등

**예시**:
```
PART 1 basic-tutorial:
- 글자 수: 1500-2000자 (초급자 최적)
- 코드 예제: 3-5개
- 코드 줄 수: 5-10줄 (짧고 명확하게)
- 대상 독자: 초급 (0-1년)
- 필수 요소: 학습 목표 (3개), 요약 (1개), 연습 문제 (5개)
- 선택 요소: 트러블슈팅 (1개), 참고 자료 (2-3개)

PART 2 technical-deep-dive:
- 글자 수: 2000-2800자 (깊이 있는 설명)
- 코드 예제: 5-8개
- 코드 줄 수: 10-20줄 (상세한 설명 포함)
- 대상 독자: 중급 (1-3년)
- 필수 요소: 학습 목표 (5개), 원리 설명 (2개), 심화 섹션 (1개), 요약
- 선택 요소: 성능 최적화 (1개), 보안 고려사항 (1개)
```

---

### 5. 형식 (Format)

**정의**: Markdown 구조, 코드 블록 언어 지정, 아이콘/이모지 사용, 표 형식

**세부 항목**:
- **Markdown 헤딩**: H1, H2, H3 사용 규칙
- **코드 블록**: 언어 지정 (```python, ```javascript, ```bash)
- **강조**: **굵게**, *기울임*, `코드`
- **목록**: 순서 있는 목록 vs 순서 없는 목록
- **표**: 복잡한 정보 정리 시 사용
- **아이콘/이모지**: 🎯 (목표), ✅ (완료), ⚠️ (주의), 💡 (팁) 등
- **인용**: > 인용문 (중요한 개념 강조)
- **링크**: 외부 문서 참조

**예시**:
```markdown
PART 1 basic-tutorial 형식:

# Chapter 1: 제목 (H1)

## 1-1: 섹션 제목 (H2)

### 학습 목표 🎯 (H3 + 이모지)

- 목표 1 (순서 없는 목록)
- 목표 2

### 개념 설명

**핵심 개념**: 프롬프트는... (굵게 강조)

> 중요: Claude Code는... (인용문으로 핵심 강조)

코드 예제:
```python
# 기본 예제 (언어 지정 필수)
def hello():
    print("Hello, World!")
```

### 표 예시

| 항목 | 설명 | 예시 |
|------|------|------|
| A | 설명 A | 예시 A |

✅ **완료**: 이제 다음 단계로... (체크 + 굵게)
```

## 📋 PART별 템플릿 상세

### PART 1: basic-tutorial (기초 튜토리얼)

**5가지 구성 요소 완전 정의**:

#### 1. 문서 구조
```
├─ 도입 (1 섹션, 1-2 단락, 각 3-4 문장)
├─ 전환 (1-2 섹션, 각 1 단락, 각 2-3 문장)
├─ 본문 섹션 1 (2-3 단락, 각 4-5 문장)
├─ 본문 섹션 2 (2-3 단락, 각 4-5 문장)
├─ 본문 섹션 3 (2-3 단락, 각 4-5 문장)
└─ 요약 (1 섹션, 1 단락, 3-4 문장)

총 섹션: 6-7개
총 단락: 11-15개
평균 문장/단락: 3-4개
```

#### 2. 문체
- **어조**: 친근하고 격려하는 톤 ("함께 배워봅시다", "천천히 따라해보세요")
- **종결어미**: "-요" (85%), "-습니다" (15%)
- **능동태 비율**: 90% ("우리는 설치합니다" > "설치됩니다")
- **문장 길이**: 평균 18단어 (짧고 명확하게)
- **용어**: 한국어 우선 (영어 병기) - "프롬프트 (Prompt)"

#### 3. 내용 전개
- **배치 순서**: 문제 제기 → 솔루션 제시 → 단계별 설명 → 확인
- **깊이**: Level 1 (기초) 70% + Level 2 (응용) 30%
- **예제 복잡도**: 5줄 → 10줄 → 15줄 (점진적 증가)
- **연결성**: 이전 장 20% 참조

#### 4. 조건
- **글자 수**: 1500-2000자
- **코드 예제**: 3-5개 (각 5-10줄)
- **대상 독자**: 초급 (0-1년)
- **필수 요소**: 학습 목표 3개, 체크포인트 3개, 요약 1개, 연습 문제 5개
- **선택 요소**: 트러블슈팅 1개

#### 5. 형식
```markdown
# Chapter N: 제목

## N-1: 섹션 제목

### 학습 목표 🎯
- 목표 1
- 목표 2
- 목표 3

### 개념 설명
**핵심**: ...

코드:
```python
# 기본 예제
```

✅ 체크포인트: ...

```

총 섹션: 10-11개
총 단락: 15-20개
평균 문장/단락: 4개

- **용어**: 실무 용어 (한국어 우선) - "컴포넌트", "API 엔드포인트"
