# Korean Typography Rules

## 맞춤법 및 띄어쓰기

**핵심 원칙** (국립국어원 표준 기반):
- **기본**: 모든 단어는 띄어 쓴다
- **조사**: 앞말에 붙여 쓴다 (예: "책을", "집에서")
- **의존 명사**: 띄어 쓴다 (예: "할 수 있다", "3개")
- **보조 용언**: 띄어 쓰되 경우에 따라 붙여 쓸 수 있다
- **전문 용어**: 붙여 쓸 수 있다 (예: "데이터베이스관리시스템")

**기술 문서 띄어쓰기 특례**:
```
✓ "웹 애플리케이션" (일반적)
✓ "웹애플리케이션" (전문 용어로 간주 시)

✓ "3 개" (원칙)
✓ "3개" (단위 명사 허용)

✓ "할 수 있다" (원칙)
✓ "할수있다" (❌ 비표준)
```

## 구두점 사용법

**마침표 (.)**:
- 문장 종결 시 필수
- 제목에는 사용하지 않음
- 목록 항목의 마침표는 일관성 유지 (모두 사용 또는 모두 생략)

**쉼표 (,)**:
```
✓ 나열: "Python, Java, JavaScript를 지원합니다"
✓ 접속: "코드를 작성하고, 테스트하고, 배포합니다"
✓ 종속: "API를 호출하면, 결과가 반환됩니다"
```

**세미콜론 (;)**:
- 한국어 기술 문서에서는 사용 자제
- 코드 예시 내에서만 사용

**괄호 ( )**:
- 한글에 붙여 쓴다: "컨테이너(Container)"
- 영문과는 띄운다: "Docker (컨테이너 기술)"

## 숫자 및 단위 표기

**숫자 표기 원칙**:
```
✓ 아라비아 숫자: 기술 데이터, 통계, 수식
  "Python 3.12", "100MB", "5개"

✓ 한글 숫자: 관용적 표현, 순서
  "첫째", "둘 중 하나", "열 번"
```

**단위 표기** (SI 단위계):
```
✓ 띄어쓰기: "10 GB", "3.5 kg"
✓ 붙여쓰기 허용: "10GB", "3.5kg"
✓ 일관성 유지 필수
```

## Markdown 포매팅 규칙

**헤더 레벨**:
```markdown
# H1: 장 제목 (책 전체에서 7-10회)
## H2: 주요 섹션 (장당 5-9회)
### H3: 하위 섹션 (필요시)
#### H4: 세부 항목 (최소한으로 사용)
```

**강조**:
- **굵게**: 핵심 용어 첫 등장 시
- *기울임*: 외래어, 책 제목
- `인라인 코드`: 변수명, 함수명, 명령어

**목록**:
```markdown
<!-- 순서 없는 목록 -->
- 항목 1
- 항목 2
  - 하위 항목 2.1
  - 하위 항목 2.2

<!-- 순서 있는 목록 -->
1. 첫 번째 단계
2. 두 번째 단계
3. 세 번째 단계
```

**코드 블록**:
````markdown
```python
# 언어 지정 필수
def hello():
    print("Hello, World!")
```
````
