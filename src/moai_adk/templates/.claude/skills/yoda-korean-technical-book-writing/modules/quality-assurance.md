# Quality Assurance Module

## 기술적 정확성

**코드 검증 프로세스**:
```bash
# 1. 모든 코드 예제 실행 테스트
pytest tests/code_examples/

# 2. 버전 호환성 확인
tox -e py39,py310,py311,py312

# 3. 보안 취약점 스캔
bandit -r examples/

# 4. 코드 스타일 검사
flake8 examples/
black --check examples/
```

**기술 리뷰 체크리스트**:
```
✓ 모든 코드 예제가 실행 가능한가?
✓ 버전별 차이점이 명시되어 있는가?
✓ 더 이상 사용되지 않는(deprecated) 기능을 사용하지 않았는가?
✓ 보안 취약점이 있는 패턴을 사용하지 않았는가?
✓ 최신 베스트 프랙티스를 따르고 있는가?
```

## 링크 및 참조 검증

**자동화 도구**:
```bash
# Markdown 링크 검사
npx markdown-link-check **/*.md

# 파이썬 코드로 링크 검증
python scripts/check_links.py
```

**체크리스트**:
```
✓ 모든 외부 링크가 HTTPS를 사용하는가?
✓ 공식 문서 링크가 최신 버전을 가리키는가?
✓ 내부 참조(장, 절 번호)가 정확한가?
✓ 이미지 경로가 올바른가?
✓ 코드 저장소 링크가 유효한가?
```

## 일관성 검사

**용어 일관성**:
```python
# 용어 사전 생성 및 검증
terms = {
    "API": ["API", "api", "Api"],  # 표준: API
    "데이터베이스": ["데이타베이스", "DB", "database"],  # 표준: 데이터베이스
}

# 비표준 용어 발견 시 경고
```

**문체 일관성**:
```
✓ 1장부터 마지막 장까지 동일한 높임말 사용
✓ 독자 호칭 통일 ("여러분", "당신", 생략 중 택1)
✓ 코드 설명 스타일 통일
✓ 연습문제 형식 통일
```

## 최종 출판 전 체크리스트

```markdown
## 콘텐츠
- [ ] 모든 장이 학습 목표와 요약을 포함하는가?
- [ ] 목차 레벨이 일관적인가?
- [ ] 코드 예제가 모두 테스트되었는가?
- [ ] 연습 문제에 해답이 제공되는가?
- [ ] 부록이 완성되었는가?

## 기술 검증
- [ ] 최신 버전 호환성 확인 (Python 3.12, Node.js 20 등)
- [ ] 모든 외부 링크 유효성 검증
- [ ] 보안 취약점 스캔 완료
- [ ] 라이선스 명시 (코드, 이미지)

## 문체 및 편집
- [ ] 맞춤법 검사 완료
- [ ] 용어 일관성 검증
- [ ] 문체 통일 확인
- [ ] 도표/이미지 품질 검토

## 메타데이터
- [ ] ISBN 확보
- [ ] 저자 소개 작성
- [ ] 책 소개 (300자, 1000자 버전)
- [ ] 키워드 선정 (SEO)
- [ ] 표지 디자인 승인

## 법률 및 윤리
- [ ] 출처 표기 완료
- [ ] 저작권 침해 검토
- [ ] 상표권 확인 (예: React®, Python™)
- [ ] 개인정보 보호 검토
```
