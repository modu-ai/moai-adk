## Core Database Patterns

### Relational Design
1. **Normalization**: Proper table structure design
2. **Indexing Strategy**: Performance optimization
3. **Query Optimization**: Efficient SQL patterns
4. **Transaction Management**: ACID compliance
5. **Data Integrity**: Constraints and validation

### NoSQL Patterns
- **Document Stores**: MongoDB, Firestore patterns
- **Key-Value Stores**: Redis, DynamoDB usage
- **Graph Databases**: Neo4j, data relationships
- **Time Series**: InfluxDB, temporal data
- **Search Engines**: Elasticsearch, full-text search
