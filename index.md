## 🎯 MoAI-ADK란?

**AI 슈퍼에이전트가 주도하는 SPEC-First TDD 개발 프레임워크**

### 핵심 원리
> **"명세(SPEC) → 테스트(TDD) → 코드 → 문서가 완벽하게 동기화되는 AI 개발 생태계"**

### 주요 특징
- **SPEC-First**: 명세가 테스트를, 테스트가 코드를 이끕니다
- **TDD 강제**: RED → GREEN → REFACTOR 사이클로 완벽한 테스트 커버리지
- **AI 슈퍼에이전트**: Alfred가 모든 개발 프로세스를 오케스트레이션합니다
- **TAG 시스템**: 코드, 테스트, 문서를 추적 가능하게 연결합니다

### 문제 해결
**"이 AI가 만든 코드를 정말 믿을 수 있을까?"** 라는 근본적인 질문에 답합니다:

- ✅ **명확한 요구사항**: SPEC으로 정의된 명확한 요구사항
- ✅ **완벽한 테스트**: TDD로 구현된 87% 이상의 테스트 커버리지
- ✅ **일관된 문서**: 코드와 동기화된 자동 문서 생성
- ✅ **컨텍스트 유지**: Alfred가 프로젝트 전체를 이해하고 관리

### 배경과 필요성

현대 AI 개발은 신뢰성 문제에 직면해 있습니다. 개발자들은 AI의 도움을 받고 싶지만, "이 코드를 정말 믿을 수 있을까?"라는 의심을 떨쳐내지 못합니다.

**MoAI-ADK는 이 문제들을 해결합니다**:
- 📋 **명세 기반 개발**: 요구사항을 명확히 정의하고 문서화합니다
- 🧪 **완벽한 테스트**: TDD로 모든 코드 경로를 테스트합니다
- 🔄 **지속적인 동기화**: 코드 변경 시 문서를 자동으로 업데이트합니다
- 🤖 **AI 에이전트 통합**: Alfred가 개발 전체를 관리합니다

### 아키텍처 개요

MoAI-ADK는 4계층 아키텍처로 구성됩니다:

1. **Commands**: CLI 인터페이스 (`moai-adk` 명령어)
2. **Sub-agents**: 전문 에이전트 (SPEC 작성, TDD 구현 등)
3. **Skills**: Claude Code 스킬 연동 및 확장성
4. **Hooks**: 개발 생태계 통합 및 자동화
