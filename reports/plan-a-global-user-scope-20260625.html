A안 — 글로벌 user-scope 구조 전환

Goal per-project 배포 모델에서 글로벌 user-scope(~/.claude + ~/.moai/config) 공통 자산 + 프로젝트 오버라이드 3-tier 모델로 전환. Claude Code scope 병합(Managed > Project > User > Plugin)을 활용해 자산 중복을 제거하고 16-locale 일관성과 머신 특이성 격리를 동시에 달성한다.

01

Milestones

5개 신규 인프라 패키지 + 검증/문서화를 순차 마일스톤으로 진행한다. 각 패키지는 기존 아키텍처(deployer · manifest · namespace · config · cli)의 자연스러운 확장 지점에 위치한다.

M1

Global Manifest Manager

~/.moai/manifest.global.json을 관리. 기존 internal/manifest(per-project .moai/manifest.json만 취급)를 글로벌 root로 확장한다. TemplateManaged/UserModified/UserCreated/Deprecated provenance 분류는 그대로 유지.

internal/manifestglobal root
M2

다중 Root Deployer

Deployer.Deploy(ctx, projectRoot, m, tmplCtx)가 글로벌 root 지원으로 확장. skip-on-user-modified 로직(deployer.go:156-172)을 글로벌 배포에도 동일 적용해 사용자 수정을 덮어쓰지 않는다.

deployer.gomulti-root
M3

Namespace 보호 확장

userOwnedScanRoots(update_namespace_protect.go:37-41, 213-235)에 글로벌 경로를 추가하고, assertNoUserOwnedNamespaceTouch가 project-relative만 검사하던 것을 글로벌로 확장한다. harness-* 및 사용자 자산을 글로벌 scope에서도 절대 삭제/수정 금지.

namespaceprotection§24 contract
M4

Config Fallback Loader

글로벌 config = 기본값, 프로젝트 config이 override하는 2-tier fallback. FindProjectRoot()(root.go:7-33)는 이미 ~/.moai/를 global state로 제외 중 — 이 원칙을 유지하며 글로벌 기본값 로딩을 얹는다.

internal/configfallback
M5

마이그레이션 경로

moai update --global 플래그 추가(update.go flags 84-96). 기존 프로젝트 → 글로벌 promote 경로를 제공한다. 1인 OSS Hybrid Trunk 정책(main-direct push)과 정합.

internal/cli--global flag
M6

검증 + 문서화

neutrality guard를 글로벌 manifest로 확장, docs-site 4-locale 갱신, README 마이그레이션 가이드 추가. 기존 프로젝트 호환성 regression 테스트로 전환 안전성을 확보한다.

validationdocs-siteregression
02

Data Flow

글로벌 user-scope 자산이 Claude Code scope 병합을 거쳐 프로젝트 scope과 합쳐지는 3-tier 구조. 배열(permissions/hooks)은 merge, 스칼라는 override된다.

실선 = 동기 scope 병합, 점선 = User scope(프로젝트가 shadow). 프로젝트 scope은 오버라이드/확장만 담당한다.
