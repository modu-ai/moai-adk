## ¬ßF Tier Justification ‚Äî Tier L (5-artifact)

**Decision: Tier L** (plan-auditor Tier L PASS threshold = 0.85). The load-bearing pillar is the **build-system change**, NOT a LOC count. Rationale:

- **PRIMARY ‚Äî Build-system change is a first-class new dependency (the L pillar).** Adding `github.com/a-h/templ` to `go.mod` + the `templ` codegen tool + a `go:generate` directive + Makefile + CI-mirror wiring is not a code edit ‚Äî it changes how the **entire tree** is built and how **CI compiles every package**. A new codegen step that `go build ./...` and the CI pipeline depend on is a cross-cutting, hard-to-reverse build infrastructure change. Cohort precedent (001/003/004/005 were Tier M) NEVER touched the build system ‚Äî they were pure code/asset edits within `internal/web`. 006 is the first cohort member to introduce a build dependency, and that build-wiring blast radius (whole-project compile + CI) is what warrants the L envelope independent of any LOC measure. The `constitutional / build-infrastructure` arm of the Tier L definition ("> 1000 LOC **or** constitutional") is satisfied by the build-system change, not by template line-count.
- **SUPPORTING ‚Äî Whole-renderer replacement + ~15 brittle Class A markup-test ports.** The ~339-line `page.html.tmpl` + its 3 `{{define}}` helpers are fully replaced by Templ components (new `*.templ` sources + generated `*_templ.go` + the `assets.go`/`handlers.go` render-call swap). The template is modest in size (339 lines, honestly stated ‚Äî NOT inflated), but the migration touches **every rendered element** and must keep ~15 markup-coupled tests green through exact-markup parity. design.md's component-tree + markup-parity mapping is load-bearing to make that 1-pass port succeed.
- **design.md + research.md are genuinely needed** (Templ component tree + helper components + the build/codegen flow + how this seeds 007's section registry; HTMX/Templ technical validation + cohort roadmap). These materially de-risk the port ‚Äî not ceremonial.

(Counter-consideration: the migration is bounded to ONE package, behavior-preserving, and the template itself is only ~339 lines ‚Äî all of which argue Tier M. The decisive factor tipping to L is the **build-system dependency** (new `go.mod` dep + codegen step the whole project's compile/CI depends on) ‚Äî a constitutional-class change Tier M does not contemplate. The 5-artifact set is the right de-risking investment for a build-infrastructure change; under-classifying as M to skip design.md would leave the component-tree port under-specified. If the auditor judges the build-system change alone insufficient for L, the honest fallback is Tier M ‚Äî but the cross-project compile/CI blast radius of a new codegen dependency is the stated reason it holds at L.)
