# Acceptance Criteria — SPEC-V3R6-TOOL-POLICY-SSOT-001

## §D. AC Matrix

### AC-TPS-001 (MUST-FIX) — SSOT artifact exists with documented schema [REQ-TPS-001, REQ-TPS-002]

**Given** the repository at plan-phase completion
**When** a maintainer inspects `.moai/config/sections/tool-policy.yaml`
**Then** the file exists, parses as valid YAML, and each entry carries the 6-field schema `{tool, args_pattern, risk_tier, decision, owner_agent, audit}` with `risk_tier ∈ {read, write, irreversible}` and `decision ∈ {allow, deny, ask}`.

**Evidence**: `test -f .moai/config/sections/tool-policy.yaml && python3 -c 'import yaml; d=yaml.safe_load(open(...)); assert all(set(e) >= {"tool","args_pattern","risk_tier","decision","owner_agent","audit"} for e in d["entries"])'`.

### AC-TPS-002 (MUST-FIX) — Seeded entries from 4 scattered sources [REQ-TPS-008]

**Given** the 4 scattered policy sources enumerated in research.md §C
**When** the seed YAML is inspected
**Then** entries are present for: (a) settings.json permissions (read-tier bulk), (b) status-transition-ownership.sh (SPEC status transitions), (c) glm-web-tooling.md PROHIBIT table (WebSearch/WebFetch/Read-image under GLM), (d) agent-common-protocol.md L162 (background-agent write deny).

**Evidence**: YAML grep for each source's representative entry; research.md §C inventory cross-checked entry-by-entry.

### AC-TPS-003 (MUST-FIX) — Codegen produces settings.json permissions block [REQ-TPS-003]

**Given** the seeded `tool-policy.yaml`
**When** `moai tool-policy build` is invoked
**Then** the `.claude/settings.json` `permissions` block is regenerated from the YAML, and the block's allow/deny lists reflect the YAML entries.

**Evidence**: `moai tool-policy build && git diff --stat .claude/settings.json` shows the permissions region changed; non-permissions regions (PATH, hooks, env) unchanged (KI-1 block-region replacement).

### AC-TPS-004 (MUST-FIX) — Round-trip semantic equivalence [REQ-TPS-004]

**Given** the codegen output (generated settings.json permissions block)
**When** a characterization test evaluates the decision for every seeded entry
**Then** the generated decision equals the YAML-declared decision for every entry (YAML → generated → equivalent behavior).

**Evidence**: `go test ./internal/config/toolpolicy/ -run TestRoundTripEquivalence` passes; the test iterates YAML entries and asserts `codegenDecision(entry) == entry.Decision`.

### AC-TPS-005 (MUST-FIX) — YAML↔settings.json drift structurally prevented [REQ-TPS-005]

**Given** the codegen generates BOTH the `.claude/settings.json` `permissions` block (enforcement surface) AND the YAML header comment (audit surface) from `tool-policy.yaml`
**When** a maintainer edits a policy rule R in the YAML (e.g., flip `Bash(git push:*)` from `allow` to `ask`) and runs `moai tool-policy build`
**Then** the generated `.claude/settings.json` `permissions` block reflects R (the entry moves from the `allow` array to the `ask` array), AND the YAML↔settings.json round-trip is decision-equivalent for every seeded entry (`codegenDecision(yamlEntry) == yamlEntry.decision`).

**Evidence (step 4 — the real deliverable, not a vague doctrine-comment check)**:
```bash
# 1. Edit YAML: flip Bash(git push:*) from allow → ask
# 2. Run codegen
moai tool-policy build
# 3. Assert enforcement surface updated: parse generated settings.json, evaluate decision for R
go test ./internal/config/toolpolicy/ -run TestDriftPrevention_YamlSettingsRoundTrip
#    The test: (a) loads YAML entries, (b) parses generated settings.json permissions block,
#    (c) for every entry asserts codegenDecision(entry) == entry.Decision,
#    (d) specifically asserts Bash(git push:*) decision == "ask" after the flip.
```

**Out of scope for this AC (§X.8)**: this AC does NOT assert that markdown doctrine (e.g., `harness-namespace-doctrine.md`) or Go enforcement code reflects R — those surfaces are not generated by this SPEC. The §24.5 markdown-vs-code drift class is explicitly NOT claimed to be preventable here.

### AC-TPS-006 (MUST-FIX) — Thin tool-policy query modeled on constitution CLI shape (NOT a schema wrapper) [REQ-TPS-006]

**Then** the query is served by a NEW thin `moai tool-policy list` subcommand (in `internal/cli/tool_policy.go`) that (a) loads `tool-policy.yaml` directly, (b) supports filter flags (`--risk-tier`, `--decision`, `--tool`) modeled on the `moai constitution list --zone` flag SHAPE, and (c) does NOT delegate to or wrap `moai constitution list` (the schemas are disjoint; wrapping is infeasible).

**Evidence**: `grep -rn "func.*ToolPolicyList\|tool-policy list" internal/cli/tool_policy.go` returns the new query command; `grep -rn "constitution" internal/cli/tool_policy.go` returns NO calls (the tool-policy query does NOT delegate to constitution — it loads the YAML directly with its own schema).

### AC-TPS-013 (MUST-FIX) — Regeneration idempotency [REQ-TPS-003]

**Given** a populated `tool-policy.yaml` and a generated `.claude/settings.json` permissions block
**When** `moai tool-policy build` is invoked TWICE in succession (with no YAML change between invocations)
**Then** the second invocation produces byte-identical output to the first — `diff` between the two generated settings.json permissions blocks is EMPTY. This is the canonical codegen-correctness property: deterministic generation with no side-effects, no timestamp drift, no ordering nondeterminism.

**Evidence**:
```bash
moai tool-policy build && cp .claude/settings.json /tmp/run1.json
moai tool-policy build && cp .claude/settings.json /tmp/run2.json
diff /tmp/run1.json /tmp/run2.json   # MUST be empty (exit 0)
```
Equivalently as a Go test: `go test ./internal/config/toolpolicy/ -run TestCodegenIdempotency` asserts `bytes.Equal(codegen(YAML), codegen(codegen(YAML)))`.

### AC-TPS-014 (MUST-FIX) — Template-side round-trip + no-template-directive-in-permissions sentinel [D7]

**Given** the codegen regenerates BOTH `.claude/settings.json` (parse-modify-serialize) AND `internal/template/templates/.claude/settings.json.tmpl` (raw-text region replacement)
**When** `moai tool-policy build` regenerates the template `.tmpl` permissions block
**Then** (a) the regenerated `.tmpl` is still a valid Go-template file (the `{{jsonEscape .SmartPATH}}` directive in the `env` block is preserved verbatim — raw-text region replacement only touches the permissions block); (b) the regenerated permissions block contains ZERO `{{` or `}}` substrings (sentinel: no template directive accidentally lands inside the permissions block); (c) the YAML→.tmpl round-trip is decision-equivalent for every seeded entry.

**Evidence**:
```bash
moai tool-policy build
# (a) template directive preserved
grep -c '{{jsonEscape .SmartPATH}}' internal/template/templates/.claude/settings.json.tmpl  # MUST be 1
# (b) no template directive inside permissions block (extract permissions region, assert zero {{ or }})
python3 -c "
import re
t = open('internal/template/templates/.claude/settings.json.tmpl').read()
m = re.search(r'\"permissions\"\s*:\s*\{', t)
region = t[m.start():]
depth = 0
for i, c in enumerate(region):
    if c == '{': depth += 1
    elif c == '}': depth -= 1
    if depth == 0: break
assert '{{' not in region[:i] and '}}' not in region[:i], 'template directive leaked into permissions block'
"
# (c) round-trip decision equivalence (same test as AC-TPS-004 but against the .tmpl target)
go test ./internal/config/toolpolicy/ -run TestTemplateRoundTripEquivalence
```

## §D.2 Edge Cases

- **EC-1**: YAML entry with `args_pattern` that is a regex containing special chars (e.g., `Bash(git push --force*)`). Codegen must escape correctly when emitting settings.json `Bash(...)` glob syntax. Test: round-trip a regex-heavy entry.
- **EC-2**: environment-gated rule (GLM backend) — the YAML must express the `Where ANTHROPIC_BASE_URL ~ api.z.ai` condition. Codegen for environment-gated rules may emit a hook matcher rather than a static permissions entry (design.md §C decides).
- **EC-3**: overlapping entries (one allow, one deny for the same tool). Resolution rule: most-specific match wins, deny wins on tie. Test: overlapping entries resolve deterministically.
- **EC-4**: empty YAML (no entries). Codegen must emit an empty (but valid) permissions block, not crash.
- **EC-5**: malformed YAML. Loader must return a clear error, not silently produce wrong policy.
