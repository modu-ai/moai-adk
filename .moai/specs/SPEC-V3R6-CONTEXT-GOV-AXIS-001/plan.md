## §A. Context

Sprint 15 harness-books application cohort, P2b. This SPEC operationalizes book2 ch8.3's signal-dilution caution + diag-05's three-paths framing as a **mechanical drift gate** on the MoAI-ADK harness control plane. The gap: moai-adk has skill-side token budgets (skillListingBudgetFraction ~1%, compaction ~25K, progressive disclosure L1/L2/L3) but **no equivalent budget or audit for the EAGERLY-loaded context** (CLAUDE.md + ~60 auto-loaded rule files + moai.md 773 LOC + MEMORY.md). Without recording the eager-vs-on-demand weight split, the project cannot answer book2's diagnostic *"have we drifted from budgeted-working-memory toward inject-first?"*

This SPEC adds the missing recording (REQ-CGA-001/002), keeps it fail-open (REQ-CGA-003), and documents a drift alarm feeding Tier-2 proposals (REQ-CGA-004/005/006).

### C.2 usage-log.jsonl schema — discovered (v2 — verified baseline 2026-06-18)

Located at `.moai/harness/usage-log.jsonl`. Verified baseline: `internal/harness/types.go:16` carries `const LogSchemaVersion = "v2"` (bumped from `v1` by SPEC-HARNESS-OUTCOME-CAPTURE-001 REQ-OC-010); the live log carries a MIX of legacy `"v1"` lines (508) and current `"v2"` lines (100). Common fields across event types AND across both `v1`/`v2` schemas:

- `timestamp` (ISO-8601 with sub-second precision)
- `event_type` ∈ {`user_prompt`, `session_stop`, `subagent_stop`, ...}
- `subject` (string, often `""` for top-level, `"unknown"` for subagent)
- `context_hash` (string, often `""`)
- `tier_increment` (integer, observed `0`)
- `schema_version` (string literal `"v1"` on legacy lines, `"v2"` on current lines)

Event-specific fields:
- `user_prompt`: `prompt_hash`, `prompt_len`, `prompt_lang`
- `session_stop`: `last_assistant_message_hash`, `last_assistant_message_len`
- `subagent_stop`: `agent_id`

**New fields to add (additive, on `v2.1` lines only — legacy `v1` AND `v2` lines render them as Go zero-value sentinel when read back)**:
- `eager_context_weight` (integer, estimated tokens or bytes — `omitempty`; absent on legacy `v1`/`v2` lines, present-on-`v2.1`-with-estimation-skipped renders as `0`)
- `on_demand_context_weight` (integer, estimated tokens or bytes — `omitempty`; same sentinel semantics)
- `weight_unit` (string, `"tokens"` or `"bytes"` — disambiguates the integer; `omitempty`)
- `schema_version` bumps from `"v2"` → `"v2.1"` on new-binary-written lines (REQ-CGA-002), so a reader can branch legacy `v1`/`v2` (pre-SPEC binary, weight fields never measured — both predate this SPEC) vs `v2.1` (new binary, estimation skipped → sentinel). NOTE: the bump is `v2` → `v2.1`, NOT `v1` → `v1.1` — `v1.1` would be a regression below the already-current `v2` baseline.
- Optionally: `eager_breakdown` (object with per-source subtotals: `claude_md`, `rules`, `output_style`, `memory_index`) — run-phase M1 decision on whether to include this or keep it aggregate (§X.6 leans aggregate).

## §G. Anti-Patterns (to avoid in run-phase)

- **AP-CGA-001** — Editing only the `.sh` wrapper (K1): the wrapper is a generated forwarder; weight-recording logic must land in the Go handler. Editing the wrapper alone produces zero effect on recorded fields and risks `moai update` overwrite.
- **AP-CGA-002** — Breaking schema backward-compat (K2): renaming/removing existing fields, or failing to default new fields to sentinel on old lines. Old lines MUST still parse.
- **AP-CGA-003** — Blocking on recording failure (K3): any `exit 2` path in the weight-recording logic violates REQ-CGA-003 and the death-spiral avoidance property.
- **AP-CGA-004** — Tier-system scope creep: rewriting Tier definitions instead of feeding Tier-2 additively (REQ-CGA-006).
- **AP-CGA-005** — Treating the drift alarm as enforcement: the alarm SURFACES proposals; it does not auto-demote rules (§X.1).
- **AP-CGA-006** — Hardcoding the Go file path: the handler file may drift; re-Glob at M1 start (K5).
- **AP-CGA-007** — Template-isolation violation: do NOT add the Context-Governance Axis § to any `internal/template/templates/` copy of a harness doc. `harness-delivery-strategy.md` is maintainer-local.
