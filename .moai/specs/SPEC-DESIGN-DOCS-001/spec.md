## Background

### 흡수 배경

`~/moai/moai-studio/.moai/design/`는 다음 4개 파일로 구성된 사람 작성 디자인 컨텍스트 번들을 운영해 왔다:

- `research.md` — 경쟁사 분석, 사용자 인사이트, 채택/기피 패턴
- `system.md` — 디자인 토큰, 타이포그래피, 스페이싱, 접근성 규칙
- `spec.md` — 정보 아키텍처, 프레임별 명세
- `pencil-plan.md` — Pencil MCP 배치 작업 계획 (선택, moai-studio에서는 `pencil-redesign-plan.md`라는 이름으로 운영되었으나 본 템플릿에서는 범용화를 위해 `pencil-plan.md`로 표준화한다)

이 구조는 `/moai design` 워크플로(Path B, code-based brand design) 실행 시 사람의 의도를 시스템에 일관되게 전달한다는 검증된 가치를 증명했다. 본 SPEC은 이 사례를 흡수하여, MoAI-ADK-Go로 초기화되는 모든 신규 프로젝트가 동일한 폴더 스캐폴딩을 `moai init` 시점에 확보하도록 템플릿 계층으로 승격한다.

### Template-First HARD 규칙 재명시

CLAUDE.local.md §2에 따라 `.claude/`, `.moai/`, `.agency/` 하위에 새 파일을 도입할 때는 항상 `internal/template/templates/` 경로에 먼저 추가하고 `make build`로 `embedded.go`를 재생성한 뒤 배포해야 한다. 본 SPEC의 모든 산출물은 `internal/template/templates/.moai/design/`에 단일 원천으로 존재한다.

### 기존 import 산출물과의 공존 전략

`moai-workflow-design-import` 스킬은 Claude Design(Path A) 핸드오프 번들을 `.moai/design/` 하위로 쓰기한다. 현재 확정된 자동 산출물은 다음과 같다:

- `tokens.json` — 디자인 토큰 (색상/타이포/스페이싱)
- `components.json` — 컴포넌트 스펙
- `assets/` — 바이너리 에셋 디렉터리
- `import-warnings.json` — 임포트 경고 로그
- `brief/BRIEF-YYYYMMDD-NNN.md` — `manager-spec` 자동 생성 브리프
- `screenshots/` — Pencil MCP `get_screenshot` 아카이브

파일 네임스페이스는 안전하게 분리된다:
- 사람 작성 파일은 `*.md` 확장자 (research/system/spec/README)
- 자동 산출물은 `*.json` 또는 전용 디렉터리 (tokens/components/import-warnings/assets/brief/screenshots)
- 예약된 파일명(§Requirements REQ-006 참조)과 사람 작성 파일명이 충돌할 여지가 없음

### 의존 관계

- `SPEC-DESIGN-CONST-AMEND-001` (FROZEN constitution 개정)이 선행되어야 한다. 현재 `.claude/rules/moai/design/constitution.md` §3 "Brand Context as Constitutional Principle"은 `.moai/project/brand/`만을 constitutional 입력으로 인정한다. `.moai/design/`를 제2의 constitutional 입력으로 승격하려면 frozen zone 개정 절차가 필요하다.

## Template Source Location

모든 템플릿 원본은 `internal/template/templates/.moai/design/` 아래 위치한다 (Template-First HARD 규칙):

| 파일 경로 | 대략 길이 | 역할 |
|---------|----------|-----|
| `internal/template/templates/.moai/design/README.md` | 30 lines | 메타 규칙, attach 정책, 예약 파일명 |
| `internal/template/templates/.moai/design/research.md.tmpl` | 50 lines | 경쟁사 분석 스켈레톤 (_TBD_) |
| `internal/template/templates/.moai/design/system.md.tmpl` | 80 lines | 디자인 토큰 스켈레톤 (_TBD_) |
| `internal/template/templates/.moai/design/spec.md.tmpl` | 60 lines | IA + 프레임 스펙 스켈레톤 (_TBD_) |

`.tmpl` 접미사 사용 이유: Go `text/template` 파이프라인을 경유한다. 현재는 정적 `_TBD_` 마커만 포함하더라도, 향후 `{{.GoBinPath}}` 등 템플릿 변수 추가 여지를 남긴다.

## Definitions

다음 용어는 Requirements 절 전반에서 해석 기준으로 사용된다.

- **User modification (REQ-005)**: 배포된 파일의 SHA-256 해시가 동일 경로의 템플릿 정본(canonical) 해시와 다르면 "사용자 수정"으로 간주한다. 공백만 다른 경우도 해시가 달라지므로 수정으로 본다.
- **Reserved filename collision (REQ-008)**: 고정 이름(`tokens.json`, `components.json`, `import-warnings.json`)은 **정확한 이름 일치**로 판정하고, 글롭 항목(`brief/BRIEF-*.md`)은 Go `filepath.Match` 규약의 패턴 매칭으로 판정한다. 사용자가 `brief/BRIEF-LOCAL.md`를 만들면 충돌로 간주된다.
- **Non-empty .moai/design/ (REQ-010)**: `.moai/design/` 하위에 **regular file(정규 파일)이 최소 1개 이상** 존재하는 상태. 숨김 파일(`.DS_Store` 등)과 빈 디렉터리만 있는 경우는 "empty"로 취급한다.
- **User-fillable section (REQ-004)**: 템플릿 렌더 결과에서 `_TBD_` 마커가 들어가는 절. 각 템플릿은 최소 1개의 `_TBD_` 마커를 포함해야 한다.

## Requirements (EARS)

각 REQ 뒤의 `Rationale` 한 줄은 해당 요구사항이 존재하는 이유를 설명한다.

1. **REQ-001** (Event-Driven): `WHEN moai init is executed on a new project, the system SHALL create the .moai/design/ directory at the project root.`
   - **Rationale**: `.moai/design/`가 존재하지 않으면 후속 템플릿 배포와 `/moai design` 워크플로가 성립하지 않는다.

2. **REQ-002** (Event-Driven): `WHEN moai init creates .moai/design/, the system SHALL deploy README.md, research.md, system.md, and spec.md rendered from their respective templates.`
   - **Rationale**: 사람 작성 디자인 컨텍스트 4종이 한 세트로 있어야 Path B 워크플로가 입력을 확보한다.

3. **REQ-003** (Event-Driven): `WHEN deploying templates, the system SHALL create empty subdirectories wireframes/ and screenshots/.`
   - **Rationale**: 사람 PNG/SVG 레퍼런스와 Pencil MCP 스크린샷 아카이브 경로를 사전 확보하여 첫 사용 시 경로 오류를 제거한다.

4. **REQ-004** (Event-Driven): `WHEN the deployed templates are opened, the system SHALL ensure each template contains at least one _TBD_ marker in every user-fillable section following the brand-voice.md pattern (.moai/project/brand/brand-voice.md).`
   - **Rationale**: `_TBD_` 마커는 사람이 채워야 할 지점을 명확히 표시하여 `evaluator-active`의 brand consistency 평가 실패를 방지한다.

5. **REQ-005** (Event-Driven): `WHEN moai update runs on an existing project, the system SHALL NOT overwrite existing files in .moai/design/ that were modified by the user (detected via SHA-256 hash mismatch against the canonical template).`
   - **Rationale**: 사용자 입력은 디자인 workflow의 constitutional 자산이므로 template 갱신이 파괴적으로 덮어쓰면 안 된다. R-1 대응.

6. **REQ-006** (Event-Driven): `WHEN README.md is rendered, it SHALL declare reserved filenames verbatim: tokens.json, components.json, import-warnings.json, brief/BRIEF-*.md.`
   - **Rationale**: 예약 파일명을 문서화해야 `moai-workflow-design-import`의 자동 산출물과 사람 작성 파일의 네임스페이스 충돌을 사전에 막을 수 있다. R-3 대응.

7. **REQ-007** (Event-Driven): `WHEN README.md is rendered, it SHALL declare auto-load priority verbatim: spec.md > system.md > research.md > pencil-plan.md.`
   - **Rationale**: `/moai design` 후속 SPEC(SPEC-DESIGN-ATTACH-001)의 attach 우선순위를 본 문서에서 단일 원천으로 선언한다.

8. **REQ-008** (Unwanted/Conditional): `IF a user-authored file collides with a reserved filename (exact-name match for fixed names; filepath.Match glob match for brief/BRIEF-*.md), THEN the system SHALL reject the file deployment at moai update time, exit non-zero, and emit an error containing the phrase "reserved filename" on stderr.`
   - **Rationale**: 충돌 허용 시 자동 산출물이 사용자 파일을 덮어쓰거나 그 반대가 발생하여 데이터 손실이 발생할 수 있다. R-3 대응.

9. **REQ-009** (Event-Driven): `WHEN make build is executed after template edits, the embedded Go file (internal/template/embedded.go) SHALL be regenerated to include the 4 new template sources.`
   - **Rationale**: `go:embed`는 재빌드 시점에만 갱신되므로 템플릿 편집 후 `make build`가 반드시 embed 목록을 동기화해야 한다.

10. **REQ-010** (Event-Driven): `WHEN moai init is invoked in a directory that already has a non-empty .moai/design/ (at least one regular file, excluding hidden files such as .DS_Store), the system SHALL log a warning to stderr and skip template deployment without modifying existing files.`
    - **Rationale**: 재실행으로 기존 자산이 파괴되는 일을 막는다. R-1 대응.

11. **REQ-011** (State-Driven): `WHILE Go template variables are absent, each .tmpl file SHALL still render correctly (pure static content is acceptable).`
    - **Rationale**: 초기 버전은 정적 콘텐츠만 포함하더라도 `.tmpl` 파이프라인을 유지하여 향후 `{{.GoBinPath}}` 등 변수 도입 여지를 남긴다.

## Acceptance Criteria

- **AC-1** (covers REQ-009 structural): `internal/template/templates/.moai/design/README.md` exists and is under 50 lines.
- **AC-2** (covers REQ-002, REQ-011): Three .tmpl files exist at `internal/template/templates/.moai/design/{research,system,spec}.md.tmpl`.
- **AC-3** (covers REQ-004): `grep -c "_TBD_" internal/template/templates/.moai/design/*.tmpl` returns at least 3 (each template has at least one _TBD_ marker).
- **AC-4** (covers REQ-001, REQ-002, REQ-003): Running `moai init test-project` in a clean tmp dir produces `.moai/design/README.md`, `.moai/design/research.md`, `.moai/design/system.md`, `.moai/design/spec.md`, plus empty `wireframes/` and `screenshots/`.
- **AC-5** (covers REQ-006): README.md contains the exact substrings `tokens.json`, `components.json`, `import-warnings.json`, `brief/BRIEF-*.md` (reserved list per REQ-006).
- **AC-6** (covers REQ-007): README.md contains the exact substring `spec.md > system.md > research.md > pencil-plan.md` (priority per REQ-007).
- **AC-7** (covers REQ-009): `go test ./internal/template/...` passes after embedded.go regeneration (template structural integrity; indirectly asserts that embedded.go lists the 4 new sources because `go test` fails on embed-list divergence from filesystem).
- **AC-8** (covers REQ-005): After running `moai init` in a tmp dir, executing `echo 'USER EDIT' >> .moai/design/README.md` and then `moai update`, the command `grep -q 'USER EDIT' .moai/design/README.md` SHALL exit 0 (user edit preserved). Template-canonical-hash compared files remain untouched.
- **AC-9** (covers REQ-008): Given a tmp project with `.moai/design/` already populated, creating a user file at `.moai/design/tokens.json` (reserved exact-match name) and running `moai update` SHALL produce exit code != 0, SHALL NOT modify `.moai/design/tokens.json`, AND SHALL emit a stderr message containing the phrase `reserved filename`. The same behavior SHALL occur for `.moai/design/brief/BRIEF-LOCAL.md` (reserved glob match).
- **AC-10** (covers REQ-010): Creating `.moai/design/stale.txt` in a tmp dir before running `moai init` SHALL cause `moai init` to exit 0 with a warning on stderr containing `.moai/design/` and `skip`. Afterwards, `stale.txt` SHALL remain unchanged AND none of `README.md`, `research.md`, `system.md`, `spec.md` SHALL be created. Directories containing only `.DS_Store` (hidden-only) SHALL be treated as empty and template deployment SHALL proceed normally.

## Scope

### IN SCOPE

- 4개 템플릿 파일 원본 정의 (README.md, research.md.tmpl, system.md.tmpl, spec.md.tmpl)
- `.moai/design/` 폴더 스캐폴딩 정의 및 `moai init` 통합 동작 명세
- 빈 하위 디렉터리 2개 (wireframes/, screenshots/) 생성 규칙
- 예약 파일명과 사람 작성 파일명의 충돌 방지 규칙 (exact match + filepath.Match)
- `moai update` 시 사용자 수정 보존 규칙 (SHA-256 기반 감지)
- `moai init` 재실행 안전 규칙 (non-empty 디렉터리 스킵)

## Exclusions (What NOT to Build)

- **`/moai design` 워크플로의 auto-load 구현**은 본 SPEC에서 다루지 않는다 (후속 SPEC-DESIGN-ATTACH-001). 본 SPEC은 폴더/템플릿 스캐폴딩만을 정의한다.
- **Pencil MCP `pencil-plan.md` 템플릿 제공**은 본 SPEC 범위 밖이다 (후속 SPEC-DESIGN-PENCIL-001). `pencil-plan.md`는 선택 파일이며 초기 배포 대상에 포함되지 않는다.
- **SPEC 파일 내용의 실제 채움**은 본 SPEC 책임이 아니다 — 최종 사용자가 `_TBD_` 마커를 채운다.
- **FROZEN constitution의 `.moai/design/` 인정**은 선행 SPEC-DESIGN-CONST-AMEND-001에서 다룬다. 본 SPEC은 해당 개정이 머지된 이후에만 `/moai run` 실행 가능하다.
- **자동 산출물(tokens.json, components.json, import-warnings.json, assets/, brief/) 생성 로직**은 `moai-workflow-design-import` 스킬 책임이며 본 SPEC과 무관하다.
- **Go 템플릿 변수(`{{.GoBinPath}}` 등) 도입**은 현 시점 범위 밖이다. 모든 `.tmpl`은 정적 렌더링만 지원한다 (REQ-011).

## Risks

- **R-1**: 기존 사용자가 `.moai/design/`에 수동으로 파일을 만들어 둔 경우 `moai update`/`moai init` 충돌 가능성
  - Mitigation: REQ-005 (SHA-256 기반 기존 파일 미덮어쓰기) + REQ-010 (비어있지 않은 `.moai/design/` 존재 시 경고 후 스킵) + AC-8, AC-10으로 검증.
- **R-2**: 템플릿이 커지면 `embedded.go` 바이너리 크기 증가로 `moai init` 초기 부팅 시간 영향
  - Mitigation: 각 템플릿을 100 lines 이하로 유지. README 30 lines, research 50 lines, system 80 lines, spec 60 lines 목표 (합계 ~220 lines, 수 KB)
