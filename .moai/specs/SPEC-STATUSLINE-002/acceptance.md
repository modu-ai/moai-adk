## AC-TU-001: 토큰 이력 기록

**시나리오**: statusline 렌더 시 세션 토큰 데이터가 캐시에 저장된다.

**전제**:
- `.moai/cache/token_history.json` 파일이 없거나 비어 있다
- Claude Code가 session_id "test-session-1", input_tokens: 1000, output_tokens: 200을 전달한다

**기댓값**:
- `.moai/cache/token_history.json`이 생성된다
- 파일에 session_id "test-session-1" 레코드가 존재한다
- `input_tokens = 1000`, `output_tokens = 200`이 기록된다

**결과**: ⬜ 미검증

## AC-TU-002: 5시간 집계

**시나리오**: 5시간 윈도우 집계가 정확히 계산된다.

**전제**:
- 캐시에 세션 A (3시간 전, 5000 토큰), 세션 B (6시간 전, 8000 토큰)가 존재한다

**기댓값**:
- 5h 집계 결과: 5000 토큰 (세션 A만 포함, 세션 B 제외)
- statusline에 `5h:5K`가 표시된다

**결과**: ⬜ 미검증

**결과**: ⬜ 미검증

## AC-TU-004: 캐시 파일 없을 때 폴백

**시나리오**: 캐시 파일 없어도 오류 없이 렌더링된다.

**전제**:
- `.moai/cache/token_history.json` 파일이 존재하지 않는다

**기댓값**:
- statusline이 오류 없이 렌더링된다
- `token_usage_5h`, `token_usage_7d` 세그먼트는 숨김 처리된다 (공백 없음)
- 다른 세그먼트(model, context 등)는 정상 표시된다

**결과**: ⬜ 미검증

## AC-TU-005: 경고 임계값 색상

**시나리오**: 임계값 초과 시 경고 색상이 적용된다.

**전제**:
- `statusline.yaml`에 `warn_5h_tokens: 50000`, `critical_5h_tokens: 100000` 설정
- 5h 집계 토큰: 75000

**기댓값**:
- 5h 값이 노란색 ANSI 색상으로 표시된다
- 7d 값은 임계값 미초과이므로 기본 색상으로 표시된다

**결과**: ⬜ 미검증

## AC-TU-006: NO_COLOR 환경 변수

**시나리오**: 색상 비활성화 시 경고 색상이 적용되지 않는다.

**전제**:
- `NO_COLOR=1` 환경 변수가 설정되어 있다
- 5h 집계가 critical 임계값을 초과한다

**기댓값**:
- ANSI 색상 코드 없이 텍스트만 출력된다
- 값 자체는 정상 표시된다

**결과**: ⬜ 미검증

## AC-TU-007: 세그먼트 비활성화

**시나리오**: `statusline.yaml`에서 세그먼트를 비활성화하면 표시되지 않는다.

- 7d 세그먼트는 정상 표시된다

**결과**: ⬜ 미검증

## AC-TU-008: 원자적 파일 쓰기

**시나리오**: 캐시 파일 쓰기 중 프로세스 중단 시 파일이 손상되지 않는다.

**전제**:

## AC-TU-009: 90일 Purge

**시나리오**: 90일 이상 된 세션 레코드가 자동으로 삭제된다.

**전제**:
- 캐시에 91일 전 세션 A, 89일 전 세션 B가 존재한다
- 마지막 Purge 시각이 25시간 이전이다

**기댓값**:
- Purge 후 세션 A가 삭제된다
- 세션 B는 유지된다
- `last_purge` 타임스탬프가 갱신된다

**결과**: ⬜ 미검증
