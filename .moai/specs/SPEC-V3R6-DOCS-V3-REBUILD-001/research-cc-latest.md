## 2. CONTEXT-MEMORY
- **context-window**: interactive "explore the context window" ŌĆö startup load: system prompt (~4.2K), auto memory (MEMORY.md), env info, MCP tool names (deferred), CLAUDE.md, skill descriptions, then per-turn reads/rules/hooks. `/context` live breakdown. Subagents isolate context. `CLAUDE_AUTOCOMPACT_PCT_OVERRIDE`. Src: /context-window (large JS page)
- **memory**: two systems ŌĆö CLAUDE.md (you write, full load every session) + auto memory (Claude writes; MEMORY.md first **200 lines OR 25KB**). CLAUDE.md hierarchy: managedŌåÆuser `~/.claude/CLAUDE.md`ŌåÆprojectŌåÆCLAUDE.local.md; rootŌåÆcwd concatenation, nested on demand. `.claude/rules/` modular + path-specific (`paths:` glob). `@path` imports (max depth 4). `/memory` lists + toggles. Auto memory requires **v2.1.59+**, stored `~/.claude/projects/<project>/memory/` (git-derived, shared across worktrees); `autoMemoryEnabled`/`CLAUDE_CODE_DISABLE_AUTO_MEMORY=1`. `InstructionsLoaded` hook debugs load. Src: /memory
- **prompt-caching**: 3 prefix layers (leastŌåÆmost volatile): System prompt, Project context (CLAUDE.md/auto memory/unscoped rules), Conversation. Exact prefix match; model + effort are cache keys. Invalidating: switch model, change effort, fast mode on, connect/disconnect MCP (only when tools in prefix), deny whole tool, `/compact`, upgrade CC. Preserving: file edits, output-style change, permission-mode change, skills/commands, `/recap`, `/rewind`, subagents. TTL 5min default; **1h TTL auto on subscription**, opt-in `ENABLE_PROMPT_CACHING_1H=1` on API. Src: /prompt-caching
- **checkpointing**: auto-tracks Claude's file edits; each user prompt = checkpoint; persists across sessions; cleaned after 30 days. `/rewind`(/checkpoint /undo) or Esc Esc empty. Menu: Restore code+conv / conv / code / Summarize from-here / up-to-here. Does NOT track bash-modified files (rm/mv/cp) or external changes; not a git replacement. "Rewind past cleared conversation" v2.1.191+. Src: /checkpointing
