## 2. Executive Summary

Three releases (v2.1.143, v2.1.144, v2.1.145) ship a mix of **hook schema extensions, worktree isolation flexibility, plugin validation tightening, permission-bypass closure, and dozens of background-session/`claude agents` polish items**. Volume is high (~90 line-items) but moai-adk-go-actionable surface is moderate (~21 items): most fixes auto-apply on upgrade.

**Headline impact on moai-adk-go (Top 5)**:

1. **Stop/SubagentStop hook input gains `background_tasks` + `session_crons` fields** (v2.1.145) — `internal/hook/subagent_stop.go` may optionally surface these in observability output.
2. **Permission-prompt bypass closed** (v2.1.145) — bare env var assignment in Bash (`FOO=bar somecmd`) was previously auto-approved when `somecmd` was non-allowlisted; now blocked. Review template Bash patterns.
3. **`worktree.bgIsolation: "none"` setting** (v2.1.143) — opt-out from background-session worktree isolation. Should be documented in `worktree-integration.md` as advanced alternative.
4. **Worktree cleanup no longer falls back to `rm -rf`** (v2.1.143) — safety win for `moai worktree done`. Note in lessons archive.
5. **Background sessions honor `permissions.defaultMode`** (v2.1.143) — local `bypassPermissions` (CLAUDE.local.md §22.1) now propagates. Behavior change for maintainer workflow.

**Recurring themes** (informational, no action):
- Heavy `claude agents` polish (10+ fixes/features across all 3 versions) → Anthropic clearly pushing background sessions to maturity
- Windows-specific UX fixes (~8 items) → moai-adk-go is macOS/Linux primary
- MCP server reliability (paginated tools/list, MIME handling, error messages) → moai uses 3+ MCP servers, benefits auto-apply

### 6.2 Permission boundary tightening (T1-2)

The bare-env-var auto-approval bypass closure (v2.1.145) is a hardening change. Audit checklist:
- Grep `internal/template/templates/.claude/agents/**` for patterns like `FOO=bar somecmd`
- Grep `internal/template/templates/.claude/hooks/**/*.sh` similarly
- Verify `permissions.allow` covers any legitimate uses

This is the most likely source of regressions in user projects after they upgrade to v2.1.145.

### 6.3 Worktree doctrine alignment (T1-7, plus T1-18 from §3)

New `worktree.bgIsolation: "none"` option (v2.1.143) plus worktree cleanup safety fix (v2.1.143) both touch the worktree subsystem. `worktree-integration.md` § Terminology Glossary should mention:
- `bgIsolation: "worktree"` (default) = current L2/L3 SPEC worktree behavior
- `bgIsolation: "none"` = opt-out for monorepo / impractical-worktree repos (user opt-in only)
- Cleanup safety: `git worktree remove` failure no longer triggers `rm -rf` fallback
