# MoAI-ADK v3.0 Workflow Redesign — 2026-05-23 Synthesis Update

## 0. Why This Update

The **v3-redesign-blueprint-2026-05-22.md** (1 day ago) defined the v3.0 architecture: 23→11 agents, 37→9 skills, 17→13 commands, 6-wave SPEC roadmap. This document **extends, does not replace** that blueprint with two new inputs:

1. **Empirical reality check** — 7 days of `/status` usage (2026-05-16 ~ 23) covering 125 main sessions + 282 subagent transcripts. The actual delegation pattern matches the blueprint's hypothesis with surgical precision (77.7% concentration in 3 agents = "handful of well-scoped agents" Anthropic pattern), but also exposes a **routing-defect** (39% catalog dead) that the blueprint did not address.
2. **Industry validation** (2026-Q1/Q2) — 3 parallel research agents pulling 65+ URLs from Anthropic engineering blog, agentskills.io spec, arXiv, ICLR 2026, Augment Code, revfactory. The blueprint's directional bets are **vindicated** by mainstream consensus, but three corrections are needed (manager-docs model tier, NOTICE.md license, plugin marketplace addition).

**Headline**: v3.0 blueprint stays intact. We add **5 amendments** and **1 new Wave** (Wave 7 — plugin marketplace packaging).

**Anthropic's verdict** (verbatim from `claude.com/blog/subagents-in-claude-code`):

> "It's tempting to define a custom subagent for everything, but **flooding Claude with options makes automatic delegation less reliable**. Most teams settle on a handful of well-scoped agents rather than a sprawling roster."

> "The description field is what Claude uses to decide when to delegate. **Be specific about the trigger conditions, not just the capability.** 'Reviews code for security issues before commits' routes better than 'security expert.'"

**Root cause of 9 dead agents** = `description:` fields written as **capability-labels** (e.g., "expert-refactoring: refactoring specialist") rather than **trigger-rules** (e.g., "Use proactively when 5+ files require the same pattern transformation, codemod, AST migration, or rename-across operation"). The auto-routing has nothing to anchor on.

### 3.2 Agent `description:` Rewrite — Trigger-Rule Pattern (NEW)

**Current MoAI pattern** (capability-label):

```yaml
description: "Refactoring specialist. Use PROACTIVELY for codemod, AST-based transformations..."
```

**Anthropic-prescribed pattern** (trigger-rule):

```yaml
description: |
  Use proactively when ANY of the following trigger conditions match:
  EN: 5+ files require same pattern transformation, rename-across, codemod,
      AST migration, bulk import alias change, dependency upgrade with breaking API
  KO: 5+ 파일 동일 패턴 변환, 일괄 rename, 코드모드, AST 마이그레이션, ...
  NOT for: single-file bug fixes, new feature implementation, ...
```

**Why this matters**: The 9 dead agents in /status data are dead BECAUSE Claude's auto-routing reads `description:` as the deciding signal. Capability labels ("X expert", "Y specialist") provide nothing to anchor on; trigger rules ("Use when X happens") give Claude routing material.

**Action**: New SPEC — `SPEC-V3R6-AGENT-DESCRIPTION-REWRITE-001` (Tier S, ~11 file edits, blocking criterion: every agent's `description:` field includes "Use when..." or "Use proactively when..." trigger phrasing in EN + 3 localized languages).
