## 권장 조치

verified Tier 1/2 중 `requires_moai_action=true`(조치필요 Y) 항목 기반의 구체적 순서화 조치:

1. **hooks-reference 문서 갱신 (Tier 1, 2.1.152)** — moai-adk가 관리하는 hook event 문서 표에 `MessageDisplay` 이벤트(+ `hookSpecificOutput.displayContent`)를 추가. 정합 근거: https://code.claude.com/docs/en/hooks
2. **SessionStart hook 문서/옵션 검토 (Tier 1, 2.1.152)** — moai-adk가 직접 deploy하는 `SessionStart` hook 문서에 신규 반환 필드 `reloadSkills`, `sessionTitle` 추가. 실제 hook 동작(`handle-session-start.sh`/`moai hook session-start`)에서 이 필드를 채택할지는 별도 결정. 정합 근거: 동일 hooks doc.
3. **skill frontmatter 스키마 문서 갱신 (Tier 1, 2.1.152)** — `.claude/rules/moai/development/skill-authoring.md`(또는 skill frontmatter 스키마 SSOT)에 `disallowed-tools` 필드 추가. 자율 skill의 `AskUserQuestion` 차단 use-case가 moai-adk 서브에이전트 boundary 정책과 시너지. 정합 근거: https://code.claude.com/docs/en/skills
4. **settings.json `agent` 필드 문서화 검토 (Tier 1, 2.1.157)** — `settings.json.tmpl` 렌더 결과 또는 settings 문서에 `agent` 키(dispatched 세션용 named subagent) 노출 여부 결정. moai-adk는 8-agent 카탈로그를 deploy하므로 이 키와의 상호작용 검토 가치 있음. 정합 근거: https://code.claude.com/docs/en/settings

> 위 4건은 모두 **문서/스키마 갱신** 수준이며, 즉시적인 Go 코드(`internal/`+`pkg/`) 변경을 강제하지 않는다. settings.json `.tmpl` 변경 여부는 `agent` 키 채택 결정에 종속.

## Umbrella SPEC 제안

verified Tier 1 중 실제 **템플릿/코드 작업**을 강제하는 건이 단독으로 존재하지 않음 — 4건 모두 "문서/스키마 문서 갱신" 수준이며, settings.json `.tmpl` 변경은 `agent` 키 **채택 결정**에 종속(현재는 문서화 검토 단계).

다만 4건이 모두 hook/skill/settings 스키마 **문서 동기화**라는 공통 cross-cutting 성격을 가지므로, 묶어서 처리할 경우 다음 경량 SPEC을 제안한다(SPEC artifact는 본 문서에서 작성하지 않음):

- **제안 디렉토리명**: `SPEC-V3R6-CC2158-SCHEMA-SYNC-001` (Tier S, docs/schema-doc only)
- **scope bullets**:
  - hooks-reference 문서에 `MessageDisplay` 이벤트 + `displayContent` 추가
  - SessionStart hook 문서에 `reloadSkills` / `sessionTitle` 반환 필드 추가
  - skill frontmatter 스키마 문서에 `disallowed-tools` 필드 추가
  - settings 문서/`.tmpl`에 `agent`(dispatched 세션 named subagent) 키 노출 여부 결정 + 문서화
  - 4-locale docs-site 동기화 의무 준수 (해당 문서가 docs-site에 노출되는 경우)

> 단, 즉각적 코드 작업이 없으므로 umbrella SPEC 생성은 **필수가 아님**. 위 4 조치는 manager-docs 단일 sync 작업으로도 처리 가능. SPEC 승격 여부는 오케스트레이터/사용자 결정.
