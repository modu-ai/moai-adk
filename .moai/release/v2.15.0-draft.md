- **BC-V3R3-001**: T2 expert agents에 Agent tool 추가 (additive, escalation depth ≤ 2 enforced)

  - 영향: 진단-only 가정한 자동화에서 동작 변경 가능

- **BC-V3R3-006** (ARCH-007 도입 후): per-agent token budget enforcement (warning-first)
  - 영향: 큰 SPEC 처리 시 자동 wave split 트리거

## 다른 프로젝트에 적용하는 방법

### 1. 업데이트

```bash
# Homebrew
brew upgrade moai-adk

# Go install
go install github.com/modu-ai/moai-adk/cmd/moai@v2.15.0

# 검증
moai version  # → v2.15.0
```

### 2. 기존 프로젝트 동기화

```bash
cd your-project/
moai update          # 11 skill PD 블록 + 7 expert tool uplift + Mobile skills 자동 배포
moai doctor          # 진단 (BC-V3R3-001~006 호환성 체크)
```

### 3. 새 프로젝트에 v2.15 패턴 활용

#### A) Mobile 프로젝트 (NEW in v2.15)

```bash
moai init my-mobile-app
cd my-mobile-app
# → CLAUDE.md에 자동으로 expert-mobile 활성화
# → /moai plan "iOS native authentication" 입력 시 expert-mobile + ios-native skill 자동 로드

# 4 strategy guide:
# - iOS native (SwiftUI)
# - Android native (Jetpack Compose)
# - React Native (cross-platform)
# - Flutter (cross-platform deep)
```

#### B) Token Budget 안전망 활용 (NEW in v2.15)

```yaml
# .moai/config/sections/runtime.yaml (자동 생성, 사용자 override 가능)
runtime:
  context_management:
    pre_clear_threshold: 0.75  # 75% 도달 → progress.md 자동 저장 + resume message
    per_agent_budget:
      manager-strategy: 60000
      expert-*: 40000
    circuit_breaker:
      stall_detection_seconds: 60
      retry_max: 3
```

#### C) Expert Escalation 활용 (NEW in v2.15)

```
사용자: /moai run SPEC-AUTH-001 (auth + db + frontend)
  → manager-tdd → expert-backend (Go API)
                  ↓ Agent() escalation (depth 1)
                  → expert-security (OWASP review during impl)
                  → expert-database (schema design)
  → manager-tdd → expert-frontend (UI)
```

이전: manager-tdd 단독 처리 (silent assumption)
이후: 도메인별 expert 자동 협업
