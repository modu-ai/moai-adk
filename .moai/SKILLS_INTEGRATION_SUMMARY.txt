================================================================================
WHAT EACH FILE CONTAINS
================================================================================

metadata.json Structure (per skill):
  - skill_id: Unique identifier (kebab-case)
  - skill_name: Human-readable name
  - version: Semantic version (2.0.0)
  - language: english (system infrastructure)
  - word_count: Content size (1200-1400 words)
  - sections: Number of teaching sections
  - context7_references: External doc links
  - agents: Which sub-agents can use this skill
  - tags: Searchable tags
  - patterns_supported: Architecture patterns covered
  - dependencies: Foundation skills required
  - created/updated: Timestamps
  - validation_status: Deployment readiness
  - spec_reference: Link to original SPEC

triggers.yaml Structure (per skill):
  - skill_id: Unique identifier
  - trigger_keywords: Words that auto-load this skill
  - trigger_contexts: Situations where skill loads
  - trigger_agents: Which agents can trigger loading
  - auto_load_conditions: Boolean rules
  - priority: Loading priority (100 = highest)
  - freedom_level: Agent discretion level

baas-skills-manifest.json Structure:
  - manifest_version: Schema version
  - project: SPEC reference
  - title: Project description
  - phase: Development phase
  - status: Production readiness
  - summary: Statistics (skills count, word count, etc.)
  - skills: Array of all skill metadata (duplicated for convenience)
  - statistics: Coverage analysis
  - deployment_checklist: Validation status of all checks
  - next_phase: Future skills to add
  - usage_guide: How to load and trigger skills
