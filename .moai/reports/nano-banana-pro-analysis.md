# 🍌 Google Nano Banana Pro 통합 분석 보고서

## 개요

**Nano Banana Pro** (정식명: **Gemini 3 Pro Image Preview**)는 Google DeepMind가 2025년 11월 20일 발표한 최신 이미지 생성 및 편집 모델입니다. 스튜디오급 디자인 품질, 정교한 텍스트 렌더링, 그리고 실시간 정보 연동을 통해 전문가급 자산(asset) 생성을 가능하게 합니다.

---

## 1. Nano Banana Pro의 핵심 특징

### 1.1 기본 사양

| 항목 | 내용 |
|------|------|
| **공식 모델명** | `gemini-3-pro-image-preview` |
| **API 기반** | Gemini API (Google AI for Developers) |
| **발표일** | 2025년 11월 20일 |
| **주요 포지셀링** | Professional asset production, 스튜디오급 디자인 |
| **지원 해상도** | 1K, 2K, 4K (최대 4096x4096) |

### 1.2 차별화된 기능들

#### 🔍 **1. 실시간 정보 연동 (Real-world Grounding)**
- Google Search 통합으로 최신 정보 기반 이미지 생성
- 현실 세계의 최신 트렌드, 이벤트, 정보 반영
- 사실 기반의 정확한 이미지 생성

#### 🧠 **2. 내장 "Thinking" 프로세스**
- 생성 전 자동으로 구성(composition)을 정제
- 다단계 사고 과정을 거친 고품질 결과물
- 프롬프트 해석 정확도 향상

#### 📝 **3. 정교한 텍스트 렌더링**
- 로고, 다이어그램, 포스터에 고품질 텍스트 생성
- 기울임, 특수 글꼴, 복잡한 레이아웃 지원
- "지금까지 경험하지 못한 정교한 제어"

#### 🎨 **4. 다중 입력 지원**
- 최대 14개 참조 이미지 동시 처리
- 6개 객체 + 5명 인물 이미지 조합
- 고급 이미지 합성 및 스타일 전이

#### 📐 **5. 유연한 이미지 종횡비**
- 1:1, 16:9, 21:9, 2:3, 3:2, 3:4, 4:3, 4:5, 5:4, 9:16 등 다양한 비율
- 용도별 맞춤형 레이아웃

#### 🔐 **6. SynthID 워터마크**
- 모든 생성 이미지에 자동 포함
- AI 생성 이미지임을 증명
- 투명성과 신뢰성 보장

## 3. API 사양 및 사용 방법

### 3.1 기본 엔드포인트

```
POST /v1beta/models/{model_id}:generateContent
```

**기본 URL**: `https://generativelanguage.googleapis.com/v1beta/models/{model_id}:generateContent`

### 3.2 요청 구조 (Request)

```json
{
  "contents": [
    {
      "parts": [
        {
          "text": "프롬프트: 생성하고자 하는 이미지 설명"
        }
      ]
    }
  ],
  "generationConfig": {
    "imageConfig": {
      "aspectRatio": "16:9",
      "imageSize": "4K"
    }
  }
}
```

### 3.3 이미지 편집 요청

```json
{
  "contents": [
    {
      "parts": [
        {
          "inline_data": {
            "mime_type": "image/png",
            "data": "BASE64_ENCODED_IMAGE_DATA"
          }
        },
        {
          "text": "편집 지시: [변경할 내용]"
        }
      ]
    }
  ]
}
```

### 3.4 응답 구조 (Response)

```json
{
  "candidates": [
    {
      "content": {
        "parts": [
          {
            "inlineData": {
              "mimeType": "image/png",
              "data": "BASE64_ENCODED_GENERATED_IMAGE"
            }
          }
        ]
      },
      "finishReason": "STOP",
      "index": 0
    }
  ]
}
```

## 4. Nano Banana Pro 베스트 프랙티스

### 4.1 프롬프트 엔지니어링 (Prompt Engineering)

#### ✅ **효과적인 프롬프트 작성**

**규칙 1: 세부적인 설명 사용**
```
❌ 나쁜 예: "강아지 사진"
✅ 좋은 예: "부드러운 황금색 털을 가진 골든 리트리버가 해변에서
공을 물고 있는 모습. 일몰의 따뜻한 빛이 털을 비추고 있고,
배경에는 파도가 있다. 전문 사진 품질, 85mm 렌즈로 촬영한 것처럼."
```

**규칙 2: 이야기식 구성**
```
원문 권장: "Describe the scene, don't just list keywords"
- 키워드 나열: ✗
- 이야기 기반 설명: ✓
```

**규칙 3: 포토그래픽 요소 명시**
- 조명 (lighting): "황금시간 빛", "드라마틱한 음영"
- 카메라 앵글: "넓은 각도", "초상화 렌즈 (85mm)", "드론 뷰"
- 분위기 (mood): "고급스러운", "장중한", "평온한"
- 색감: "따뜻한 톤", "파스텔", "비브런트"

#### 4.2 이미지 편집 프롬프트 전략

**원칙: 명확하고 구체적으로**

```
예시 1) 객체 추가
❌ "로고 추가"
✅ "회사 로고를 셔츠의 왼쪽 가슴 부분에 자연스럽게
인쇄된 것처럼 배치하되, 셔츠의 천 주름을 따라 변형되도록"

예시 2) 스타일 전이
❌ "반 고흐 스타일로 변경"
✅ "제공된 도시 거리 사진을 빈센트 반 고흐의 '별이 빛나는 밤'
스타일로 변환하되, 소용돌이 치는 붓질(impasto), 깊은 파란색과
밝은 노란색의 극적인 팔레트를 사용하고, 건물과 자동차의
원래 구성은 유지할 것"

예시 3) 요소 제거
❌ "배경 제거"
✅ "사진의 배경에 있는 사람들을 제거하고,
그 자리에 자연스러운 벽돌 벽 텍스처로 채울 것"
```

### 4.3 멀티턴 대화 활용

**Thought Signature 관리 (자동 처리)**
- 공식 SDK 사용 시: 자동으로 처리됨
- REST API 직접 사용 시: 응답에서 받은 `thought_signature` 값을 다음 요청에 그대로 포함

```python
# Python SDK 예제
from google import genai

client = genai.Client(api_key="YOUR_API_KEY")

# 첫 번째 생성
response1 = client.models.generate_content(
    model="gemini-3-pro-image-preview",
    contents=[{
        "parts": [{
            "text": "프롬프트 1: 아름다운 풍경"
        }]
    }]
)

# 이미지를 받고 수정 요청
image_data = response1.candidates[0].content.parts[0].inline_data.data

response2 = client.models.generate_content(
    model="gemini-3-pro-image-preview",
    contents=[{
        "parts": [
            {
                "inline_data": {
                    "mime_type": "image/png",
                    "data": image_data
                }
            },
            {
                "text": "하늘을 더 파랗게 만들어줄 수 있을까?"
            }
        ]
    }]
)
```

### 4.4 안전성 및 제약사항

#### ⚠️ **주의사항**

1. **이미지 업로드 권한**
   - 업로드하는 이미지의 사용 권리 확인 필수
   - 저작권 침해 방지

2. **검색 기반 생성**
   - 이미지 기반 검색 결과는 제외됨
   - 순수 Google Search 결과만 활용

3. **언어 지원**
   - 최적 언어: en, es-mx, ja-jp, zh-cn, hi-in
   - 다른 언어 사용 시 품질 저하 가능

4. **인물 생성 규제**
   - 실제 인물로 오인될 수 있는 이미지에 규제
   - 명확한 일러스트레이션 스타일 권장

### 4.5 성능 최적화

#### 토큰 효율

| 요청 유형 | 예상 토큰 사용량 | 처리 시간 |
|----------|---------|---------|
| Text-to-Image (1K) | ~1,000-2,000 | 10-20초 |
| Text-to-Image (4K) | ~4,000-8,000 | 30-60초 |
| Image-to-Image Editing | ~2,000-3,000 | 15-30초 |
| Multi-turn conversation | 누적 | 점진적 증가 |

#### 최적화 전략

1. **배치 처리**
   - 여러 요청을 한 번에 처리 (API 한계 내)
   - 처리 시간 효율화

2. **해상도 최적화**
   - 용도에 맞게 1K, 2K, 4K 선택
   - 4K는 인쇄물, 고가용도만 사용

3. **캐싱**
   - 자주 사용되는 프롬프트 템플릿 저장
   - 반복 요청 최소화

4. **에러 처리**
   - 안전성 필터 위반 시 재시도 전략 수립
   - 타임아웃 설정 (30초 이상)
