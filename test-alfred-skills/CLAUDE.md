# Alfred Skills Test Suite

**4개 Alfred 스킬 기능 검증 프로젝트**

> **문서 언어**: 한국어
> **프로젝트 소유자**: GOOS
> **설정**: `.moai/config.json`
>
> **참고**: 이 프로젝트는 Alfred 스킬들의 기능적 완성도를 검증하기 위한 테스트 환경입니다.

---

## 🎯 테스트 목표

이 테스트 프로젝트는 다음 4개 핵심 Alfred 스킬들이 제대로 구현되었는지 검증합니다:

1. **moai-alfred-clone-pattern** - 복잡한 다단계 작업의 자율적 위임
2. **moai-alfred-personas** - 사용자 전문성 수준에 따른 적응형 커뮤니케이션
3. **moai-alfred-reporting** - 표준화된 보고서 생성 및 출력 형식 관리
4. **moai-alfred-doc-management** - 문서 위치 규칙 및 결정 트리

---

## 🧪 테스트 시나리오

### 스킬 1: Clone Pattern 테스트
- **시나리오**: 대규모 프로젝트 리팩토링 요청
- **검증 항목**: Alfred가 복잡성을 인식하고 클론 패턴을 적용하는지
- **예상 결과**: 자율적 클론 생성, 병렬 처리, 학습 메모리 저장

### 스킬 2: Personas 테스트
- **시나리오**: 다양한 전문성 수준의 사용자와 상호작용
- **검증 항목**: Alfred가 사용자 전문성을 탐지하고 적절한 페르소나를 선택하는지
- **예상 결과**: 상세한 설명(초급) 또는 간결한 가이드(고급)

### 스킬 3: Reporting 테스트
- **시나리오**: 작업 결과 보고서 생성 요청
- **검증 항목**: 화면 출력과 내부 문서의 형식을 올바르게 구분하는지
- **예상 결과**: 화면에는 일반 텍스트, 파일에는 마크다운 형식

### 스킬 4: Doc Management 테스트
- **시나리오**: 다양한 종류의 문서 생성 요청
- **검증 항목**: 문서를 올바른 위치에 저장하고 금지된 패턴을 피하는지
- **예상 결과**: .moai/ 계층 구조에 문서 저장, 루트 레벨 문서 생성 금지

---

## 📋 테스트 실행 절차

### 1단계: 개별 스킬 기능 테스트
각 스킬의 핵심 기능을 개별적으로 검증합니다.

### 2단계: 통합 워크플로우 테스트
4개 스킬이 함께 작동하는 복합 시나리오를 테스트합니다.

### 3단계: 결과 검증 및 보고
예상 출력과 실제 출력을 비교하고 최종 보고서를 생성합니다.

## 📝 테스트 로그

모든 테스트 실행은 상세한 로그로 기록됩니다:

- **실행 시간**: 각 테스트의 시작 및 종료 시간
- **입력 데이터**: 사용된 테스트 시나리오 및 파라미터
- **실제 출력**: Alfred 스킬들이 생성한 결과
- **성공 여부**: 예상 결과와의 일치 여부
- **오류 분석**: 실패한 경우의 원인 분석

## 🎩 Alfred 역할
