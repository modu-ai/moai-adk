# Clone Pattern Skill Test Scenario 1

## 테스트 시나리오: 대규모 프로젝트 마이그레이션

### 사용자 요청 (복잡성: 높음)
"이 전체 프로젝트를 구버전에서 신버전으로 마이그레이션해줘. 모든 파일의 import 경로를 업데이트하고, 설정 파일들을 새로운 포맷으로 변환하고, 테스트 코드도 수정하고, 문서도 업데이트해줘."

### 예상 복잡성 지표
- **예상 단계 수**: 8단계 이상
- **영향 파일 수**: 100개 이상
- **도메인 전문성 필요**: 없음 (구조적 변경 작업)
- **불확실성**: 높음 (마이그레이션 과정에서 예기치 않은 문제 발생 가능)
- **병렬 처리 가능**: 있음 (파일별 병렬 처리)

### Clone Pattern 적용 예상 조건
✅ 5단계 이상 작업 (8단계 예상)
✅ 100개 이상 파일 영향
✅ 도메인 전문가 불필요
✅ 높은 불확실성
✅ 병렬 처리 이득

### 예상 Alfred 동작
1. **복잡성 분석**: 작업 복잡도 평가
2. **패턴 결정**: Clone Pattern 적용 결정
3. **클론 생성**: 자율적 마이그레이션 클론 생성
4. **실행**: 클론이 독립적으로 마이그레이션 실행
5. **학습**: 경험을 clone-memory.json에 저장

### 예상 출력
- 화면: "마이그레이션 복잡도 감지. 클론 패턴을 적용합니다."
- 진행 상황: 병렬 처리 진행률 표시
- 최종 결과: 마이그레이션 완료 보고 및 학습 저장 확인
