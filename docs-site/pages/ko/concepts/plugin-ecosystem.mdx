# Plugin 생태계: 5가지 전문 플러그인

MoAI-ADK의 확장성 계층입니다. 5개의 전문 플러그인으로 **풀스택 개발, DevOps 배포, 기술 블로깅**을 자동화합니다.

## 개요

**목적**: Alfred SuperAgent를 확장 → 도메인별 특화 기능 제공

**생태계 규모**:
- **5개 플러그인**: Backend, DevOps, Frontend, UI/UX, Technical Blog
- **26개 Agent**: 전문가 수준의 협력 에이전트
- **25개 Skills**: 도메인별 모범 사례 집합
- **15개 Commands**: 플러그인별 사용자 진입점

---

## 플러그인 구조

```
Plugin Architecture (공통 패턴):
┌────────────────────────────────────────┐
│ Plugin.json (메타데이터)                 │
├────────────────────────────────────────┤
│ - name: 플러그인명                      │
│ - version: 버전 (1.0.0-dev)             │
│ - description: 한줄 설명                │
│ - commands: [3-4개 엔트리 포인트]       │
│ - agents: [3-7개 전문 Agent]           │
│ - skills: [3-6개 도메인 Skill]         │
│ - mcpServers: SaaS 통합 (옵션)         │
└────────────────────────────────────────┘
          ↓
┌────────────────────────────────────────┐
│ Commands/ (사용자 진입점)                │
├────────────────────────────────────────┤
│ - `/init-fastapi` - 프로젝트 생성       │
│ - `/db-setup` - 데이터베이스 설정       │
│ - `/resource-crud` - CRUD 자동 생성     │
└────────────────────────────────────────┘
          ↓
┌────────────────────────────────────────┐
│ Agents/ (전문가 협력)                   │
├────────────────────────────────────────┤
│ - 아키텍트 (Sonnet) - 전략 수립         │
│ - 개발자 (Haiku) - 코드 구현            │
│ - 전문가 (Haiku) - 특화 기능            │
│ - 협력자 (Haiku) - 통합                 │
└────────────────────────────────────────┘
          ↓
┌────────────────────────────────────────┐
│ Skills/ (모범 사례)                     │
├────────────────────────────────────────┤
│ - 도메인 패턴                           │
│ - 언어 모범 사례                        │
│ - 프레임워크 가이드                     │
│ - SaaS 서비스 통합                      │
└────────────────────────────────────────┘
```

## 1️⃣ Backend Plugin

**용도**: FastAPI 백엔드 + SQLAlchemy ORM + Alembic 마이그레이션

### 플러그인 정보

| 항목 | 값 |
|------|-----|
| **ID** | `moai-plugin-backend` |
| **Version** | 1.0.0-dev |
| **Language** | Python 3.13+ |
| **Framework** | FastAPI 0.120.2 |
| **ORM** | SQLAlchemy 2.0 |
| **Package Manager** | uv 0.9.3 |

### 4개 Specialist Agents

```
1. API Designer (Sonnet)
   - RESTful API 설계 및 스키마 정의
   - OpenAPI 3.1 스펙 생성
   - 엔드포인트 패턴 설계

2. Backend Architect (Sonnet)
   - 전체 아키텍처 설계
   - 계층별 책임 분리
   - 의존성 주입 패턴

3. Database Expert (Haiku)
   - SQLAlchemy 스키마 설계
   - Alembic 마이그레이션 작성
   - 성능 최적화

4. FastAPI Specialist (Haiku)
   - 엔드포인트 구현
   - Pydantic 모델 생성
   - 에러 처리 및 검증
```

### 4개 Skills

- **moai-domain-backend** - Backend 아키텍처 패턴
- **moai-lang-python** - Python 3.13+ 모범 사례
- **moai-domain-database** - 데이터베이스 설계 최적화
- **moai-lang-fastapi-patterns** - FastAPI 모범 사례

### Commands

```bash
# FastAPI 프로젝트 초기화
/init-fastapi

# PostgreSQL 또는 MySQL 데이터베이스 설정
/db-setup

# SPEC에서 CRUD 엔드포인트 자동 생성
/resource-crud
```

### 사용 예시

```bash
# 예제 1: 새 FastAPI 프로젝트 생성
$ /init-fastapi "e-commerce API"
  → FastAPI 프로젝트 구조 생성
  → pyproject.toml + uv.lock 생성
  → 기본 라우터 및 미들웨어 설정

# 예제 2: User 리소스 CRUD 생성
$ /resource-crud "User 리소스"
  → models/user.py (SQLAlchemy)
  → schemas/user.py (Pydantic)
  → routes/users.py (FastAPI 엔드포인트)
  → 테스트 케이스 자동 생성

# 예제 3: 데이터베이스 마이그레이션
$ /db-setup "PostgreSQL 초기 설정"
  → Alembic 초기화
  → 첫 번째 마이그레이션 생성
  → 데이터베이스 연결 테스트
```

## 2️⃣ DevOps Plugin

**용도**: 다중 클라우드 배포 (Vercel, Supabase, Render)

### 플러그인 정보

| 항목 | 값 |
|------|-----|
| **ID** | `moai-plugin-devops` |
| **Version** | 1.0.0-dev |
| **Platforms** | Vercel, Supabase, Render |
| **CI/CD** | GitHub Actions |
| **Infrastructure** | Docker, Kubernetes 준비 |

### 4개 Specialist Agents

```
1. Deployment Strategist (Sonnet)
   - 배포 전략 및 파이프라인 설계
   - 다중 환경 관리 (dev, staging, prod)
   - CI/CD 자동화 계획

2. Vercel Specialist (Haiku)
   - Next.js 배포 최적화
   - 엣지 함수 설정
   - 환경변수 관리

3. Supabase Specialist (Haiku)
   - PostgreSQL 데이터베이스 설정
   - 실시간 구독 구성
   - 보안 정책 설정

4. Render Specialist (Haiku)
   - Python/Node.js 서비스 배포
   - 환경 변수 및 시크릿 관리
   - 배포 후 검증
```

### 6개 Skills

- **moai-domain-devops** - DevOps 아키텍처
- **moai-saas-vercel-mcp** - Vercel 플랫폼 통합
- **moai-saas-supabase-mcp** - Supabase 통합
- **moai-saas-render-mcp** - Render 배포
- **moai-domain-backend** - 백엔드 배포 패턴
- **moai-domain-frontend** - 프론트엔드 배포 패턴

### Commands

```bash
# 배포 전략 수립
/plan-deployment

# Vercel에 배포
/deploy-vercel

# Supabase 데이터베이스 초기화
/setup-supabase

# Render에 백엔드 배포
/deploy-render
```

### 배포 워크플로우

```
사용자: /deploy-vercel "Next.js 앱"
       ↓
Deployment Strategist:
  - Vercel 프로젝트 설정 분석
  - 환경변수 요구사항 식별
  - CI/CD 파이프라인 생성
       ↓
Vercel Specialist:
  - vercel.json 생성
  - 환경변수 설정
  - 배포 트리거
       ↓
자동 배포:
  - GitHub에서 빌드
  - Vercel CDN에 배포
  - 미리보기 URL 생성
  - 프로덕션 배포 (main)
```

### 지원하는 플랫폼

| Platform | 용도 | 지원 기능 |
|----------|------|---------|
| **Vercel** | Frontend 배포 | Next.js, React, SvelteKit |
| **Supabase** | Database + Auth | PostgreSQL, Real-time, Auth |
| **Render** | 백엔드 & 크론 | Python, Node.js, Worker |

   - Next.js 프로젝트 구조 설계
   - 라우팅 전략
   - 상태 관리 패턴

3. Performance Optimizer (Haiku)
   - Core Web Vitals 최적화
   - 이미지 최적화
   - 번들 크기 감소

4. TypeScript Specialist (Haiku)
   - 타입 안전성 강화
   - 런타임 검증

### 5개 Skills

- **moai-domain-frontend** - Frontend 아키텍처
- **moai-lang-typescript** - TypeScript 모범 사례
- **moai-design-shadcn-ui** - shadcn/ui 컴포넌트

### Commands

```bash
# Next.js 프로젝트 초기화
/init-nextjs

# 컴포넌트 생성
/generate-component

# 페이지 생성 (라우팅 자동)
/generate-page

# 성능 분석 및 최적화
/analyze-performance
```

### 사용 예시

```bash
# 예제 1: Dashboard 애플리케이션 생성
$ /init-nextjs "Dashboard App"
  → Next.js 15 프로젝트 구조
  → TypeScript 설정 (strict mode)
  → TailwindCSS 설정
  → ESLint + Prettier

# 예제 2: Button 컴포넌트 생성
$ /generate-component "Button with icon"
  → components/ui/button.tsx (shadcn/ui)
  → variants 설정 (size, variant)
  → 타입 정의
  → Storybook 스토리

# 예제 3: 사용자 대시보드 페이지 생성
$ /generate-page "dashboard/users"
  → app/dashboard/users/page.tsx
  → DataTable 컴포넌트
  → 필터링 및 검색
  → 서버 액션 통합
```

| **Features** | SEO, Code Examples, Images, llms.txt |

### 7개 Specialist Agents

```
1. Technical Content Strategist (Sonnet)
   - 콘텐츠 전략 수립
   - 대상 청중 분석
   - 구조 계획

2. Technical Writer (Haiku)
   - 블로그 포스트 작성
   - 템플릿 기반 구조
   - 명확한 설명

3. Code Example Curator (Haiku)
   - 실행 가능한 코드 예제 생성
   - 다중 언어 지원
   - 검증

4. Visual Content Designer (Haiku)
   - 이미지 프롬프트 생성
   - 다이어그램 및 차트
   - OG 이미지

5. SEO & Discoverability Specialist (Haiku)
   - 메타 태그 생성
   - 해시태그 최적화
   - llms.txt 업데이트
   - AI 검색 가능성

6. Markdown Formatter (Haiku)
   - markdownlint 검증
   - 자동 수정
   - 품질 보증

7. Template Workflow Coordinator (Sonnet)
   - 템플릿 선택 자동화
   - Agent 협력 조율
   - 최종 조립
```

### 5가지 블로그 템플릿

| Template | 용도 | 구조 |
|----------|------|------|

| **How-to** | 실용 가이드 | Goal → Prerequisites → Steps → Verification |
| **Announcement** | 기능 발표 | What's New → Features → Getting Started |
| **Comparison** | 비교 분석 | Overview → Options → Matrix → Conclusion |

## 통합 워크플로우 예시

### 시나리오: 완전한 풀스택 애플리케이션 구축

```
Step 1: 백엔드 초기화 (Backend Plugin)
$ /init-fastapi "E-commerce API"
  → FastAPI + SQLAlchemy + Alembic 설정

Step 2: 데이터베이스 설정 (Backend + DevOps Plugin)
$ /db-setup "PostgreSQL"
  → Supabase와 통합
$ /setup-supabase
  → 데이터베이스 초기화

Step 3: CRUD API 생성 (Backend Plugin)
$ /resource-crud "Product"
  → Product 모델, 스키마, 엔드포인트 자동 생성

Step 4: 프론트엔드 생성 (Frontend Plugin)
$ /init-nextjs "E-commerce Dashboard"
  → Next.js 15 + TypeScript + TailwindCSS

Step 5: 컴포넌트 설계 (UI/UX Plugin)
$ /design-system "E-commerce Theme"
  → 색상, 타이포그래피, 컴포넌트 시스템

Step 6: 페이지 생성 (Frontend Plugin)
$ /generate-page "products/list"
  → Product DataTable 자동 생성
  → 백엔드 API 통합

Step 7: Vercel 배포 (DevOps Plugin)
$ /deploy-vercel "E-commerce Dashboard"
  → 자동 배포 파이프라인

Step 8: 기술 블로그 작성 (Blog Plugin)
$ /blog-write "E-commerce 구축 가이드"
  → Tutorial 템플릿
  → 7개 Specialist Agent 병렬 실행
  → 완성된 포스트 + SEO 최적화
```
