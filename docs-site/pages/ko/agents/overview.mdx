# Agent 시스템: 12개 Specialist Agent

Alfred SuperAgent는 12개의 **전문가 Agent**로 구성된 팀을 관리합니다. 각 Agent는 특화된 도메인에서 뛰어난 성능을 발휘하며, 함께 협력하여 SPEC-First TDD 워크플로우를 실현합니다.

## Agent 시스템 아키텍처

```
Alfred SuperAgent (Command Router)
├─ /alfred:0-project
│  └─ 🗂️ project-manager (프로젝트 초기화)
│
├─ /alfred:1-plan
│  └─ 🏗️ spec-builder (SPEC 작성)
│
├─ /alfred:2-run
│  ├─ 🔬 tdd-implementer (구현)
│  ├─ 🛡️ quality-gate (품질 검증)
│  └─ 📊 implementation-planner (계획)
│
└─ /alfred:3-sync
   ├─ 📖 doc-syncer (문서 동기화)
   ├─ 🚀 git-manager (Git 관리)
   └─ 🛡️ quality-gate (품질 검증)

Support Agents (모든 워크플로우에서 호출 가능):
├─ 🏷️ tag-agent (TAG 검증/관리)
├─ ✅ trust-checker (TRUST 5 검증)
├─ 🐛 debug-helper (에러 분석/해결)
├─ ⚙️ cc-manager (Claude Code 설정)
└─ 🛠️ skill-factory (Skill 생성/유지보수)
```

---

## 12개 Agent 분류

### 📊 분류 기준

| 분류 | 개수 | Agent | 모델 |
|------|------|-------|------|
| **Core Workflow** | 4개 | spec-builder, tdd-implementer, doc-syncer, git-manager | Sonnet×2 + Haiku×2 |
| **Setup & Planning** | 2개 | project-manager, implementation-planner | Sonnet×2 |
| **Verification** | 2개 | quality-gate, trust-checker | Haiku×2 |
| **Support** | 4개 | tag-agent, debug-helper, cc-manager, skill-factory | Haiku×4 |

## Core Workflow Agents (4개)

### 1️⃣ spec-builder 🏗️

**모델**: Sonnet (고급 추론)
**호출**: `/alfred:1-plan` (Step 1-2)
**역할**: SPEC 문서 작성 및 요구사항 분석

**책임**:
- 사용자 요구사항을 EARS 형식으로 변환
- SPEC 문서 작성 (spec.md, plan.md, acceptance.md)
- 아키텍처 설계 및 TAG 체인 구성
- 품질 검증 및 동기화 준비

**특징**:
- 대화형 질문으로 요구사항 명확화
- EARS 5가지 패턴 자동 적용
- TAG 자동 생성 및 추적
- 수정 워크플로우 지원

---

### 2️⃣ tdd-implementer 🔬

**모델**: Sonnet (고급 추론)
**호출**: `/alfred:2-run` (Step 3)
**역할**: TDD RED-GREEN-REFACTOR 사이클 실행

**책임**:
- RED: 실패하는 테스트 작성
- GREEN: 최소 구현으로 테스트 통과
- REFACTOR: 코드 정리, 커버리지 85%+ 달성
- TAG 체인 유지 및 커밋 자동화

**특징**:
- 4단계 TDD 사이클 자동 실행
- 단위 테스트 자동 작성
- 코드 품질 자동 검증 (ruff, mypy)
- 변수명, 함수명 영어로 자동화
- 커밋 메시지 자동 생성

---

### 3️⃣ doc-syncer 📖

**모델**: Haiku (빠른 실행)
**호출**: `/alfred:3-sync` (Step 4)
**역할**: 코드 변경사항에 따른 문서 동기화

**책임**:
- Living Document 철학 구현
- TAG 기반 변경사항 추적
- 자동 API 문서 생성
- 문서-코드 일관성 검증
- TRUST 5 원칙 적용

---

### 4️⃣ git-manager 🚀

**모델**: Haiku (빠른 실행)
**호출**: `/alfred:2-run`, `/alfred:3-sync` (Step 4)
**역할**: Git 워크플로우 자동화

**책임**:
- Feature 브랜치 자동 생성
- 커밋 메시지 자동 작성
- Pull Request 자동 생성 및 관리
- PR 리뷰어 자동 할당 (Team mode)
- Merge 및 배포 자동화

---

## Setup & Planning Agents (2개)

### 5️⃣ project-manager 📋

**모델**: Sonnet (고급 추론)
**호출**: `/alfred:0-project`
**역할**: 프로젝트 초기화 및 전략 수립

**책임**:
- 프로젝트 메타데이터 수집
- product.md, structure.md, tech.md 생성
- .moai/config.json 자동 생성
- 프로젝트 초기 구조 설정

---

### 6️⃣ implementation-planner 📊

**모델**: Sonnet (고급 추론)
**호출**: `/alfred:2-run` (Phase 1)
**역할**: 구현 전략 수립 및 계획

**책임**:
- SPEC 분석 및 구현 순서 결정
- Task 분해 및 의존성 매핑
- 병렬 실행 최적화
- 위험 요소 식별

---

## Verification Agents (2개)

### 7️⃣ quality-gate 🛡️

**모델**: Haiku (빠른 실행)
**호출**: `/alfred:2-run` (Phase 2.5)
**역할**: TRUST 5 원칙 자동 검증

**책임**:
- 테스트 커버리지 검증 (85%+)
- 코드 가독성 검증 (≤300줄)
- 타입 안전성 검증 (100% type hints)
- 보안 검증 및 TAG 무결성 확인

---

### 8️⃣ trust-checker ✅

**모델**: Haiku
**호출**: 모든 워크플로우 (선택적)
**역할**: TRUST 5 원칙 심화 검증

**책임**:
- 테스트 설계 품질 검증
- 코드 유지보수성 분석
- SOLID 원칙 준수 확인
- 보안 정책 준수 검증

---

## Support Agents (4개)

### 9️⃣ tag-agent 🏷️

**모델**: Haiku
**호출**: 모든 Agent (자동)
**역할**: TAG 시스템 관리 및 검증

### 🔟 debug-helper 🐛

**모델**: Haiku
**호출**: 에러 발생 시
**역할**: 에러 분석 및 해결

### 1️⃣1️⃣ cc-manager ⚙️

**모델**: Haiku
**호출**: `/alfred:0-project` (설정 시)
**역할**: Claude Code 설정 관리

### 1️⃣2️⃣ skill-factory 🛠️

**모델**: Haiku
**호출**: 새 Skill 생성 시
**역할**: Skill 개발 및 유지보수

## Agent 협력 패턴

### Sequential (순차 실행)

```
spec-builder → tdd-implementer → doc-syncer + git-manager
```

### Parallel (병렬 실행)

```
tdd-implementer ↔ quality-gate (동시 검증)
```

### Support (지원)

```
에러 발생 → debug-helper 분석 → 수정 제안
```

| Agent | 모델 | 선택 이유 |
|-------|------|---------|
| spec-builder | Sonnet | 복잡한 요구사항 분석 |

## 다국어 지원

모든 Agent는 사용자의 설정 언어로 입력을 받고, **같은 언어로** 문서를 생성합니다.

```
사용자 (한국어) → spec-builder → 한국어 SPEC + 영어 @TAG
```
