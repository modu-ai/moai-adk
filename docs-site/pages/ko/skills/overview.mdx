# Skills 시스템: 75개 전문 지식 카프셀

Alfred의 12개 Agent는 **75개의 Specialist Skills**로 지원됩니다. Skills는 **Progressive Disclosure** 패턴으로 설계되어, 필요할 때만 로드되어 속도와 정확성을 함께 제공합니다.

## Skills 아키텍처: 6-Tier System

```
┌─────────────────────────────────────────────────────┐
│ Foundation Tier (6개 Skills)                         │
│ - 핵심 개념: EARS, Git, Specs, Tags, Trust, Langs   │
├─────────────────────────────────────────────────────┤
│ Essentials Tier (4개 Skills)                        │
│ - 일상 업무: Debug, Perf, Refactor, Review         │
├─────────────────────────────────────────────────────┤
│ Alfred Tier (7개 Skills)                           │
│ - Alfred 기능: Ears-authoring, Git-workflow, ...   │
├─────────────────────────────────────────────────────┤
│ Domain Tier (10개 Skills)                          │
│ - 도메인 지식: Backend, Frontend, Database, ...    │
├─────────────────────────────────────────────────────┤
│ Language Tier (18개 Skills)                        │
│ - 프로그래밍 언어: Python, TypeScript, Go, Rust... │
├─────────────────────────────────────────────────────┤
│ Framework/Design/Content/Ops Tier (30개 Skills)   │
│ - Framework: React, Next.js, FastAPI               │
│ - Design: TailwindCSS, Figma, shadcn/ui           │
│ - Content: Blog, SEO, Markdown, Image             │
│ - Ops: SaaS (Vercel, Supabase, Render)            │
└─────────────────────────────────────────────────────┘
```

## 1️⃣ Foundation Tier (6개)

### 핵심 개념 (코어 지식)

```
moai-foundation-ears
├─ EARS 5가지 패턴 (UBIQUITOUS, EVENT-DRIVEN, STATE-DRIVEN, OPTIONAL, UNWANTED)
├─ 요구사항 문법 및 검증
└─ 테스트 케이스 자동 생성

moai-foundation-specs
├─ SPEC 메타데이터 (ID, version, status, 의존성)
├─ SPEC 작성 템플릿
└─ YAML frontmatter 검증

moai-foundation-tags
├─ @TAG 명명 규칙 (4-Core: @SPEC, @TEST, @CODE, @DOC)
├─ TAG 체인 무결성 검증
└─ TAG 통계 및 분석

moai-foundation-trust
├─ TRUST 5 원칙 (Test, Readable, Unified, Secured, Trackable)
├─ 품질 메트릭 정의
└─ 검증 체크리스트

moai-foundation-git
├─ GitFlow 전략 (Feature/Release/Hotfix)
├─ 커밋 메시지 표준
└─ 브랜치 보호 규칙

moai-foundation-langs
├─ 언어별 감지 (Python, TypeScript, Go, Rust...)
├─ 프로젝트 구조 인식
└─ 언어별 도구 체인
```

## 🔄 Progressive Disclosure Pattern

Skills는 **필요할 때만 로드**됩니다:

```
사용자: "/alfred:2-run SPEC-AUTH-001"
       ↓
tdd-implementer (필요한 Skills 결정):
       ↓
1️⃣ Core Skills 로드 (자동)
   └─ Skill("moai-essentials-debug")
       ↓
2️⃣ 조건부 Skills 로드
   ├─ Skill("moai-lang-python") [Python 프로젝트 감지]
   ├─ Skill("moai-essentials-refactor") [REFACTOR 단계 진입]
   └─ Skill("moai-essentials-perf") [성능 요구사항 있음]
       ↓
3️⃣ 명시적 호출 Skills
   └─ Skill("moai-domain-backend") [사용자가 요청]
```

## 📋 Skill 사용 가이드

### 올바른 Skill 호출 (Explicit Invocation)

```python
# ✅ 올바른 예시
Skill("moai-lang-python")
Skill("moai-foundation-specs")
Skill("moai-alfred-ears-authoring")

# ❌ 잘못된 예시 (키워드 기반)
# Skill 자동 감지 불가능
# Skill 이름 오류
```

### Skill 선택 기준

```
1. 현재 작업 단계 확인
   → Setup: Foundation + Domain + Language
   → TDD: Essentials + Framework
   → Verify: TRUST + Domain

2. 프로젝트 타입 확인
   → Backend: moai-domain-backend + moai-lang-python
   → Frontend: moai-domain-frontend + moai-lang-typescript
   → Full-stack: 모두 필요

3. 명시적 로드
   → Skill("명확한-이름") 호출
```

## 🎯 Best Practices

### ✅ Do's

- 필요한 Skill만 명시적으로 호출
- Skill 이름 정확하게 입력
- 조건부 로직으로 성능 최적화

### ❌ Don'ts

- 모든 Skill을 한 번에 로드 (성능 저하)
- Skill 이름 추측 (실패)
- 키워드 기반 자동 감지 (작동하지 않음)

## 📚 Skills 발견 및 학습

### Skill 찾기

```bash
# 사용 가능한 모든 Skills 확인
$ ls -la .claude/skills/

# 특정 Skill 문서 읽기
$ cat .claude/skills/moai-lang-python/SKILL.md
```

### Progressive Disclosure 구조

각 Skill은 3가지 섹션으로 구성:

1. **SKILL.md** (개요)
   - 개념 설명, 사용 시기, 핵심 내용

2. **examples.md** (실제 사용)
   - 코드 예제, 워크플로우, 사용 케이스

3. **reference.md** (상세 참고)
   - API 문서, 명령어, 고급 주제
