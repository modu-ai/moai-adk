# MoAI-ADK 주요 개념

## 1. SPEC-First 개발

모든 개발은 명확한 요구사항 정의부터 시작합니다.

### EARS 문법

**E**asy **A**pproach to **R**equirements **S**yntax

```
WHEN [조건]
WHILE [상태]
WHERE [제약]
THE SYSTEM SHALL [동작]
```

### 예시

```markdown
WHEN 유효한 이메일과 비밀번호가 제공되면,
THE SYSTEM SHALL JWT 토큰을 발급해야 한다.

WHERE 이메일 형식이 유효하지 않으면,
THE SYSTEM SHALL 400 에러를 반환해야 한다.
```

## 2. Test-Driven Development (TDD)

RED-GREEN-REFACTOR 사이클을 따릅니다.

### RED Phase
- 실패하는 테스트 작성
- 요구사항을 테스트 코드로 표현

### GREEN Phase
- 테스트를 통과하는 최소 구현
- YAGNI 원칙 준수

### REFACTOR Phase
- 코드 품질 개선
- 기능 변경 없이 구조 개선

## 3. @TAG 시스템

모든 SPEC, TEST, CODE, DOC을 연결합니다.

### TAG 형식

```
@SPEC:AUTH-001    # SPEC 요구사항
@TEST:AUTH-001    # 테스트 코드
@CODE:AUTH-001    # 구현 코드
@DOC:AUTH-001     # 문서
```

### 추적 가능성

```bash
# 특정 SPEC과 관련된 모든 코드 찾기
rg "@SPEC:AUTH-001"

# 결과
# src/auth.py:15:    # @CODE:AUTH-001
# tests/test_auth.py:8:    # @TEST:AUTH-001
# docs/api.md:42:    <!-- @DOC:AUTH-001 -->
```

## 4. Alfred SuperAgent

19개 AI 에이전트가 협력하는 팀입니다.

### 핵심 에이전트

- **spec-builder**: SPEC 작성
- **tdd-implementer**: TDD 구현
- **doc-syncer**: 문서 동기화
- **git-manager**: Git 작업

### Skills 시스템

55개 재사용 가능 지식 캡슐:
- Foundation: EARS, TRUST, TAG
- Essentials: TDD, Refactor, Debug
- Alfred: Commands, Context, Memory

## 5. TRUST 5 원칙

코드 품질 기준입니다.

### T - Test First
테스트를 먼저 작성합니다.

### R - Readable
읽기 쉬운 코드를 작성합니다.

### U - Unified
통일된 코딩 스타일을 유지합니다.

### S - Secured
보안을 고려합니다.

### T - Trackable
@TAG로 추적 가능하게 합니다.
