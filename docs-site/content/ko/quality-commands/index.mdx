# 품질 명령어

MoAI-ADK의 코드 품질 관리 명령어를 소개합니다.

<Callout type="info">
품질 명령어는 코드의 **리뷰, 테스트 커버리지, E2E 테스트, 아키텍처 분석**에 특화된 명령어입니다. 코드 품질을 체계적으로 관리하고 개선할 수 있습니다.
</Callout>

## 명령어 비교

| 명령어 | 목적 | 실행 방식 | 사용 시점 |
|--------|------|-----------|-----------|
| `/moai review` | 코드 리뷰 | 보안/성능/품질/UX 4가지 관점 분석 | PR 전 코드 리뷰가 필요할 때 |
| `/moai coverage` | 커버리지 분석 | 테스트 갭 분석 및 테스트 생성 | 테스트 커버리지를 높이고 싶을 때 |
| `/moai e2e` | E2E 테스트 | 브라우저 자동화 테스트 생성/실행 | 사용자 플로우를 검증하고 싶을 때 |
| `/moai codemaps` | 아키텍처 문서 | 코드베이스 구조 분석 및 문서화 | 프로젝트 아키텍처를 파악하고 싶을 때 |

## 명령어 관계도

```mermaid
flowchart TD
    A[품질 명령어] --> B[분석 명령어]
    A --> C[테스트 명령어]

    B --> D["/moai review<br/>코드 리뷰"]
    B --> E["/moai codemaps<br/>아키텍처 문서"]
    C --> F["/moai coverage<br/>커버리지 분석"]
    C --> G["/moai e2e<br/>E2E 테스트"]

    D -->|이슈 발견 시| H["/moai fix"]
    F -->|테스트 부족 시| G
```

<Callout type="tip">
**어떤 명령어를 써야 할지 모르겠다면?**

- 코드 품질을 전반적으로 점검하고 싶다면 → `/moai review`
- 테스트가 부족한 부분을 찾아 보강하고 싶다면 → `/moai coverage`
- 사용자 관점에서 앱이 잘 동작하는지 확인하고 싶다면 → `/moai e2e`
- 프로젝트 구조를 이해하고 문서화하고 싶다면 → `/moai codemaps`
</Callout>
