## Prerequisites

Before starting the database workflow, you need:

1. `.moai/project/product.md` and `.moai/project/tech.md` files generated by `/moai project`
2. A supported database engine (PostgreSQL, MySQL, SQLite, MongoDB, etc.)
3. An ORM or query builder (GORM, sqlc, Prisma, SQLAlchemy, ActiveRecord, etc.)
4. A migration tool (golang-migrate, Flyway, Liquibase, Alembic, etc.)

## Step-by-Step Initialization Guide

### Step 1: Verify Project Metadata

First, confirm that required files exist by running `/moai project` if you haven't already:

```bash
ls -la .moai/project/
# These files should exist:
# - product.md
# - tech.md
# - structure.md
```

If these files don't exist, run `/moai project` first.

### Step 2: Initialize Database Metadata

Now run the `/moai db init` command:

```bash
/moai db init
```

### Step 3: Answer Interview Questions

MoAI will ask you 4 interactive questions:

1. **Database Engine** — Your database (PostgreSQL, MySQL, SQLite, MongoDB, etc.)
2. **ORM/Query Builder** — Data access layer tool
3. **Multi-tenancy Strategy** — Single schema, schema-per-tenant, database-per-tenant, or none
4. **Migration Tool** — Schema change management tool

Select the appropriate option for each question.

### Step 4: Review Generated Files

After initialization, these files are created in `.moai/project/db/`:

```
.moai/project/db/
├── README.md              # DB section overview
├── schema.md              # Auto-generated table registry
├── erd.mmd                # Entity relationship diagram
├── migrations.md          # Migration file index
├── rls-policies.md        # Row-level security rules (Supabase/Postgres)
├── queries.md             # Common query library
└── seed-data.md           # Seed data patterns
```

File descriptions:

- `schema.md` — Automatically documents all tables, columns, data types, and constraints
- `erd.mmd` — Visualizes table relationships in Mermaid syntax
- `migrations.md` — Timeline of applied migrations
- `queries.md` — Common query examples for AI agents to reference

### Step 5: Write Your First Migration and Sync

Add a new migration file to your project. For example, with Go/golang-migrate:

```bash
# Create migration file in db/migrations/
touch db/migrations/001_create_users_table.sql
```

Write your migration, then refresh the schema documentation:

```bash
/moai db refresh
```

This command:
- Scans all migration files
- Adds new table information to schema.md
- Updates the erd.mmd diagram
- Refreshes the migrations.md timeline

### Step 6: Verify for Drift (Optional)

Check if there's any drift:

```bash
/moai db verify
```

Results:

- `Schema documentation is in sync` — Migrations and documentation match
- Drift report output — Detailed differences shown (exit code: 1)

## Troubleshooting

### "Missing prerequisite files" Error

If `.moai/project/product.md` and `.moai/project/tech.md` don't exist:

```bash
/moai project
```

Run this command first to generate project metadata.

### Migration Files Not Recognized

Check that your project's language and migration tool are correctly detected:

```bash
cat .moai/config/sections/language.yaml
```

Check the `language` field. If needed, you can manually specify `migration_patterns` in `.moai/config/sections/db.yaml`.

### Automatic Sync Not Working

Verify that the PostToolUse hook is correctly registered:

```bash
grep -A5 "PostToolUse" .claude/settings.json
```

If the hook is missing, run `/moai db init` again or manually register it in `.claude/settings.json`.
