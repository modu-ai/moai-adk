MoAI-ADK のデータベースワークフローは、プロジェクトのデータベースメタデータを集中管理します。`/moai db` コマンドを使用して、マイグレーションファイルをスキャンし、スキーマドキュメントを自動生成し、ドリフトを検出できます。

## 主な機能

- **対話型初期化** — `/moai db init` を実行してデータベースエンジン、ORM、マイグレーションツールを選択し、メタデータテンプレートを自動生成
- **自動同期** — PostToolUse フックがマイグレーションファイルの変更を自動検出してドキュメントをリフレッシュ
- **ドリフト検出** — `/moai db verify` でスキーマドキュメントとマイグレーションファイル間の矛盾を検出
- **16 言語サポート** — Go、Python、TypeScript、Rust、Java、Kotlin、C#、Ruby、PHP、Elixir、C++、Scala、R、Flutter、Swift

## 4つのサブコマンド

```bash
/moai db init      # 対話型インタビューでDB メタデータを初期化
/moai db refresh   # マイグレーションファイルを再スキャンしスキーマドキュメントを再生成
/moai db verify    # ドリフトをチェック (読み取り専用)
/moai db list      # すべてのテーブルをMarkdown テーブルで表示
```

## 使用場面

- 新規プロジェクト開始時のデータベースメタデータ設定
- マイグレーションファイル追加/編集後のドキュメント自動更新
- チームメンバーとのスキーマ状態共有
- スキーマドキュメントと実際のマイグレーション状態の一貫性検証
