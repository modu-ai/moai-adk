# `@DOC:PORTAL-LINK-001: 온라인 문서 포털 연계 가이드`

## 개요

이 문서는 MoAI-ADK 프로젝트의 GitHub README.ko.md와 온라인 문서 포털(https://adk.mo.ai.kr) 간의 연계 기능에 대한 사용자 가이드입니다.

## 문서 연계 구조

### 1. GitHub 저장소 (진입점)

**주요 역할**:

- 사용자의 첫 진입점 역할
- 프로젝트 소스 코드와 기본 문서 제공
- 온라인 문서 포털로의 안내

**문서 구조**:

- `README.ko.md` - 핵심 개요와 빠른 시작 가이드
- `docs/split/` - 상세 문서 목록
- 온라인 문서 포털 연계 링크

### 2. 온라인 문서 포털 (상세 정보)

**주요 역할**:

- 상세한 가이드와 참고 자료 제공
- 최신 정보와 상호작용 기능
- 검색 및 네비게이션 최적화

**기능 특징**:

- 실시간 업데이트되는 콘텐츠
- 고급 검색 기능
- 사용자 피드백 시스템
- 자동 링크 검증

## 사용자 경험 최적화

### 초보자 경로

1. GitHub README에서 기본 개념 이해
2. README의 빠른 시작 가이드 따라가기
3. 상세 가이드를 위해 온라인 문서 포털로 이동

### 중급자 경로

1. GitHub README에서 핵심 기능 파악
2. 온라인 문서 포털에서 심화 내용 학습
3. API 참고 문서 및 튜토리얼 활용

### 고급자 경로

1. GitHub 저장소에서 소스 코드 확인
2. 온라인 문서 포털에서 아키텍처 문서 참조
3. 고급 기능 및 확장 기능 학습

## 자동 검증 시스템

### 링크 검증

- **주기적 검사**: 자동으로 모든 링크의 유효성 검증
- **실시간 알림**: 깨진 링크 즉시 감지 및 알림
- **보고서 생성**: 자동화된 링크 무결성 보고서

### 콘텐츠 동기화

- **버전 정보 동기화**: GitHub와 온라인 문서 간 버전 정보 자동 동기화
- **릴리즈 노트**: 새 버전 릴리스 시 자동으로 릴리즈 노트 생성
- **변경사항 추적**: 주요 변경사항의 자동 추적 및 공유

## 문제 해결

### 링크가 작동하지 않을 때

1. **자동 검증 보고서 확인**: 생성된 보고서에서 링크 상태 확인
2. **임시 접속**: 일시적인 네트워크 문제일 수 있음
3. **문의**: GitHub Issues에 문의하여 보고

### 정보 불일치 발견 시

1. **GitHub 이슈 생성**: 발견된 불일치에 대해 이슈 작성
2. **문서 개선 제안**: 구체적인 개선 방안 제안
3. **동기화 요청**: 정보 업데이트를 위한 요청

## 기술적 구현

### 검증 시스템

- **비동기 처리**: 여러 링크를 동시에 검증하여 성능 최적화
- **타임아웃 설정**: 10초 타임아웃으로 장애 링크 빠르게 식별
- **에러 핸들링**: 다양한 네트워크 오류에 대한 대응

### 보고서 생성

- **통계 정보**: 응답 시간, 성공률 등 상세 통계
- **오류 분류**: 오류 유형별 분류 및 원인 분석
- **개선 제안**: 기반 데이터 기반의 개선 방안 제시

## 추천 사용 패턴

### 최적의 사용 방법

1. **GitHub 저장소**: 프로젝트 개요와 빠른 시작 가이드 확인
2. **온라인 문서 포털**: 상세한 학습과 심화 내용 탐색
3. **검색 기능 활용**: 원하는 정보 빠르게 찾기
4. **피드백 참여**: 사용자 경험 개선을 위한 피드백 제공

### 생산성 향상 팁

- **키워드 검색**: 정확한 키워드로 검색하여 시간 절약
- **북마크 활용**: 자주 방문하는 페이지 북마크 설정
- **문서 피드백**: 유용한 정보에 대해 긍정적인 평가

## 미래 계획

### 기능 확장

- **다국어 지원**: 여러 언어 문서 연계 강화
- **개인화 추천**: 사용자 행동 기반 문서 추천 시스템
- **AI 보조**: AI 기반의 질의응답 및 검색 기능

### 품질 향상

- **더 정밀한 검증**: 더 정교한 링크 검증 알고리즘 도입
- **성능 최적화**: 검색 및 네비게이션 속도 개선
- **사용자 경험 개선**: 지속적인 사용자 피드백 반영
