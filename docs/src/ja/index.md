# MoAI-ADK: AI駆動型SPEC-First TDD開発フレームワーク

> **AIの力を借りて、信頼性が高く、保守しやすいソフトウェアを構築してください。** 要件からドキュメントまで、すべての成果物は完全に追跡され、自動的にテストされ、常に同期されます。

---

## :bullseye: 解決する問題

### AI駆動型開発における6つの主要な問題

| 問題 | 影響 |
|------|------|
| **曖昧な要件** | 開発者が時間の40%を要件の明確化に費やしている |
| **テスト不足** | テストされていないコードからの本番環境バグ |
| **ドキュメントの不同期** | ドキュメントが実装と一致していない |
| **失われたコンテキスト** | チームメンバー間での説明の繰り返し |
| **影響分析が不可能** | 要件変更時に影響を受けるコードを特定できない |
| **品質の不一貫性** | 手動QAでのエッジケースの見落とし |

### MoAI-ADKの解決策

✅ **SPEC-First**: コーディング前に明確な要件を定義
✅ **保証されたテスト**: 自動TDDで87%+のテストカバレッジを達成
✅ **生きたドキュメント**: 自動同期、決して古くならない
✅ **継続的なコンテキスト**: Alfredがプロジェクト履歴とパターンを記憶
✅ **完全なトレーサビリティ**: `@TAG`システムですべてのアーティファクトを接続
✅ **品質自動化**: TRUST 5原則を自動的に実施

## ⚡ 主な機能

### 1. SPEC-First開発
- **EARS形式の仕様**: 構造化された、曖昧でない要件
- **実装前の明確化**: コストのかかった手直しを防止
- **自動トレーサビリティ**: 要件 → コード → テストをリンク

### 2. 自動化されたTDDワークフロー
- **RED → GREEN → REFACTOR**サイクル管理
- **テスト優先保証**: テストなしのコードはない
- **87%+カバレッジ**: 体系的テストで達成

### 3. Alfredスーパーエージェント
- **19の専門AI エージェント** (spec-builder、tdd-implementer、doc-syncerなど)
- **93のプロダクションレディスキル** (すべての開発領域をカバー)
- **適応学習**: プロジェクトパターンから自動学習
- **スマートコンテキスト管理**: プロジェクト構造と依存関係を理解

### 4. @TAGシステム (完全なトレーサビリティ)

すべてのアーティファクトを接続するトレーサビリティシステム：

```
@SPEC:AUTH-001 (要件)
    ↓
@TEST:AUTH-001 (テスト)
    ↓
@CODE:AUTH-001:SERVICE (実装)
    ↓
@DOC:AUTH-001 (ドキュメント)
```

### 5. 生きたドキュメント
- **リアルタイム同期**: コードとドキュメントは常に整列
- **手動更新なし**: 自動生成
- **多言語サポート**: Python、TypeScript、Go、Rustなど
- **自動図表生成**: コード構造から自動作成

### 6. 品質保証
- **TRUST 5原則**: テスト優先、読みやすい、統一、セキュア、追跡可能
- **自動化された品質ゲート** (リント、型チェック、セキュリティスキャン)
- **プリコミット検証**: コミット前に違反を防止
- **包括的なレポート**: 実行可能なメトリクス

## 🚀 クイックスタート

### インストール (推奨: uvツール)

```bash
# uvツールを使用してmoai-adkをグローバルにインストール
uv tool install moai-adk

# インストール確認
moai-adk --version

# 新しいプロジェクトを初期化
moai-adk init my-awesome-project
cd my-awesome-project
```

### プロジェクト設定 (必須)

インストール後、プロジェクトを設定する必要があります:

```bash
# プロジェクトメタデータと環境を初期化
/alfred:0-project
```

### 5分のクイックスタート

```bash
# 1. 新機能を計画 - SPEC自動生成
/alfred:1-plan "JWTトークンを使用したユーザー認証"

# 2. TDD実行 - テスト → 実装 → リファクタリングを自動化
/alfred:2-run SPEC-AUTH-001

# 3. ドキュメントを同期し、品質を検証
/alfred:3-sync
```

**結果**: 明確な要件 → テスト優先実装 → 自動ドキュメント → 品質保証がすべて完了！
