"
MoAI-ADK를 최대한 활용하기 위한 검증된 패턴과 권장 사례들을 정리했습니다. 팀 규모별, 프로젝트 유형별로 최적化的인 전략을 제공합니다.
"
## 🎯 핵심 원칙
"
### SPEC-First 개발
"
**철학**: 모든 구현은 명확한 요구사항에서 시작되어야 합니다.
"
**실천 방법**:
1. **요구사항 구체화**: 모호한 요청 피하기
   ```bash
   # ❌ 나쁜 예시
   "/alfred:1-plan 사용자 기능 추가"
"
   # ✅ 좋은 예시
   "/alfred:1-plan React TypeScript로 소셜 로그인 기능 구현
   - Google, GitHub OAuth2 지원
   - JWT 토큰 24시간 유효
   - 에러 처리 및 로딩 상태 표시"
   ```
"
2. **MVP 먼저**: 핵심 기능부터 시작하여 점진적 확장
3. **수용 기준 정의**: 완료 조건을 명확하게 명시
"
### TRUST 5 원칙 준수
"
**Test First**: 항상 테스트부터 작성
- 테스트 커버리지 85%+ 목표
- RED → GREEN → REFACTOR 엄격 준수
- 통합 테스트와 단위 테스트 균형
"
**Readable**: 가독성 있는 코드 유지
- 일관된 명명 규칙
- 의미 있는 함수/변수 이름
- 적절한 주석과 문서
"
**Unified**: 일관된 스타일 적용
- 팀 코딩 컨벤션 준수
- 일관된 아키텍처 패턴
- 표준화된 에러 처리
"
**Secured**: 보안 규칙 준수
- OWASP Top 10 주기적 검토
- 입력값 검증 및 출력 인코딩
- 최소 권한 원칙 적용
"
**Trackable**: 모든 변경 추적 가능
- 의미 있는 커밋 메시지
- @TAG 체인 무결성 유지
- 문서와 코드 동기화
"
### 자동화-First 접근
"
**반복 작업 자동화**:
```bash
# 매일 실행할 정기 작업
moai-adk doctor
moai-adk status
moai-adk backup
"
# 정기 업데이트 (월간)
moai-adk update
```

## 👥 팀 규모별 전략
"
### 1인 팀 (Personal)
"
**특징**: 빠른 의사결정, 유연한 워크플로우
"
**권장 설정**:
```json
// .moai/config.json
{
  "git_strategy": {
    "mode": "personal",
    "auto_delete_branches": false
  },
  "hooks": {
    "timeout_seconds": 10  // 더 긴 타임아웃
  }
}
```
"
**워크플로우**:
1. 아이디어 → `/alfred:1-plan`
2. 바로 구현 → `/alfred:2-run`
3. 문서화 → `/alfred:3-sync`
4. 개인 회고 → `/alfred:9-feedback`
"
**팁**:
- 자주 커밋하여 작업 단위 작게 유지
- `/alfred:?`를 적극적으로 활용하여 기억 상기
- 개인 학습 노트를 `.moai/memory/`에 관리
"
### 소규모 팀 (2-5인)
"
**특징**: 협업 필요, 코드 리뷰 중요
"
**권장 설정**:
```json
{
  "git_strategy": {
    "mode": "team",
    "base_branch": "develop",
    "auto_delete_branches": true
  },
  "tags": {
    "scan_policy": "strict"
  }
}
```
"
**워크플로우**:
1. **팀 기획**: `/alfred:1-plan`으로 요구사항 정리
2. **Draft PR**: 자동 생성된 PR로 팀 리뷰
3. **분할 개발**: 기능별로 개발자 분배
4. **통합 테스트**: `/alfred:2-run`으로 품질 보증
5. **동기화**: `/alfred:3-sync`로 문서 정리
"
**협업 팁**:
- **PR 템플릿** 사용: 일관된 리뷰 프로세스
- **데일리 스탠드업**: `moai-adk status` 결과 공유
- **페어 프로그래밍**: 복잡한 기능은 함께 `/alfred:2-run`
"
### 대규모 팀 (6인+)
"
**특징**: 전문화, 계층적 구조, 엄격한 프로세스
"
**권장 설정**:
```json
{
  "git_strategy": {
    "mode": "team",
    "base_branch": "main",
    "require_approvals": 2
  },
  "github": {
    "spec_git_workflow": "feature_branch",
    "auto_delete_branches": true
  }
}
```
"
**조직 구조**:
- **아키텍트**: `/alfred:1-plan` 주도, 시스템 설계
- **Tech Lead**: 코드 리뷰, 품질 게이트
- **개발자**: 기능 구현, 테스트 작성
- **DevOps**: CI/CD, 인프라 관리
"
**고급 워크플로우**:
1. **Epic 계획**: 대규모 기능을 여러 SPEC으로 분할
2. **전문가 배정**: 도메인 전문가 에이전트 적극 활용
3. **병렬 개발**: 여러 feature 브랜치 동시 진행
4. **통합 테스트**: 전체 시스템 검증
5. **릴리스**: `/alfred:release-new`로 공식 배포
"
## 🏗️ 프로젝트 유형별 전략
"
### 웹 애플리케이션
"
**기술 스택**:
- Frontend: `moai-lang-typescript` + `moai-domain-frontend`
- Backend: `moai-lang-python` + `moai-domain-backend`
- Database: `moai-domain-database`
"
**권장 패턴**:
```bash
# API 개발 순서
/alfred:1-plan "API 엔드포인트 설계"  # 먼저 API 계약 정의
/alfred:2-run SPEC-001               # Backend 구현
/alfred:2-run SPEC-002               # Frontend 구현
/alfred:3-sync                       # API 문서 업데이트
```
"
**디렉터리 구조**:
```
src/
├── frontend/          # React/Next.js
├── backend/           # FastAPI/Flask
├── shared/           # 공용 타입/유틸리티
└── tests/            # 통합 테스트
```
"
### 데이터 분석 프로젝트
"
**기술 스택**:
- Python: `moai-lang-python` (pandas, numpy focus)
- ML: `moai-domain-ml`
- 데이터베이스: `moai-domain-database`
"
**특별한 고려사항**:
- **데이터 버전 관리**: DVC나 MLflow 통합
- **실험 추적**: `.moai/experiments/` 디렉터리
- **모델 레지스트리**: `.moai/models/` 관리
"
**워크플로우 추가**:
```bash
# 데이터 분석 프로젝트 사이클
/alfred:1-plan "데이터 전처리 파이프라인"
/alfred:2-run SPEC-001               # ETL 개발
/alfred:3-sync                       # 데이터 문서화
# 실험 기록은 별도 프로세스
```

5. **릴리스**: `/alfred:release-new`로 자동화

## 📚 코드 품질 관리
"
### TDD 실천 전략
"
**RED 단계**:
```python
# 실패하는 테스트 먼저 작성
def test_user_authentication():
    # Given: 사용자가 존재
    user = create_test_user(email="test@example.com")
"
    # When: 잘못된 비밀번호로 로그인 시도
    result = auth_service.login("test@example.com", "wrong_pass")
"
    # Then: 인증 실패
    assert result.success is False
    assert result.error == "Invalid credentials"
```
"
**GREEN 단계**:
- 최소한의 코드로 테스트 통과
- 테스트를 깨뜨리지 않는 선에서 구현
"
**REFACTOR 단계**:
- 코드 중복 제거
- 가독성 개선
- 성능 최적화
"

- [ ] @TAG 체인 완전
- [ ] 문서 업데이트
- [ ] 보안 검증 통과
- [ ] 성능 영향 평가

"

## 🔧 시스템 운영
"
### 정기적인 유지보수
"
**일일 (5분)**:
```bash
moai-adk status      # 현재 상태 확인
git status           # 변경 사항 확인
```
"
**주간 (15분)**:
```bash
moai-adk backup      # 백업 생성
moai-adk doctor      # 환경 점검
find . -name "*.log" -mtime +7 -delete  # 로그 정리
```
"
**월간 (30분)**:
```bash
moai-adk update      # 템플릿 동기화
# .moai/reports/ 리뷰
# 팀 회고 및 개선 계획
```
"
### 모니터링 설정
"
**핵심 메트릭**:
- 개발 속도: 주당 SPEC 완료 수
- 품질: 테스트 커버리지, 버그 수
- 효율성: `/alfred:2-run` 실행 시간
- 협업: PR 리뷰 시간, 병합 충돌 수
"
**알림 설정**:
```bash
# 품질 게이트 실패 시
curl -X POST "webhook-url" \
  -H "Content-Type: application/json" \
  -d '{"message": "Quality gate failed"}'
"
# 매일 상태 리포트
moai-adk status | mail -s "Daily Status" team@company.com
```
