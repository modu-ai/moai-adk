"
Alfred는 상황과 목적에 맞는 다양한 출력 스타일을 사용하여 사용자에게 최적의 정보 전달 방식을 제공합니다. `.claude/output-styles/alfred/*.md`에 정의된 스타일 템플릿을 기반으로 일관되고 전문적인 응답을 생성합니다.
"
## 출력 스타일 종류
"
| 스타일 | 목적 | 주요 사용 시나리오 | 톤 특징 |
| ------ | ---- | ----------------- | -------- |
| **report** | 공식 보고서 | `/alfred:3-sync`, `/alfred:release-new` | 객관적, 구조화, 데이터 중심 |
| **retrospective** | 회고 및 학습 | `/alfred:9-feedback`, 프로젝트 종료 | 성찰적, 건설적, 행동 지향 |
| **tutorial** | 단계별 가이드 | `/alfred:0-project`, 기능 학습 | 교육적, 친절, 실용적 |
| **analysis** | 심층 분석 | 코드 리뷰, 아키텍처 분석 | 분석적, 근거 기반, 비판적 |
| **executive** | 경영 보고 | 프로젝트 요약, 진행률 보고 | 간결, 핵심 중심, 의사결정 지원 |
| **technical** | 기술 문서 | API 문서, 구현 가이드 | 정확, 상세, 전문가용 |
"
## 주요 출력 스타일 상세 설명
"
### `report` - 공식 보고서 스타일
"
**목적**: 프로젝트 현황, 진행률, 결과를 객관적으로 전달
"
**구조**:
```
# [보고서 제목]
"
## 📊 핵심 지표
- 완료된 SPEC: X개
- 진행 중인 작업: Y개
- 테스트 커버리지: Z%
"
## 📋 상세 현황
### 완료된 작업
- [SPEC-001] 사용자 인증 기능 ✅
- [SPEC-002] 데이터베이스 스키마 ✅
"
### 진행 중인 작업
- [SPEC-003] API 엔드포인트 (80%)
- [SPEC-004] 프론트엔드 UI (40%)
"
## 🚨 이슈 및 리스크
- [높음] 보안 검증 필요
- [중간] 성능 테스트 대기
"
## 📈 다음 단계
1. 보안 검증 완료
2. 통합 테스트 실행
3. 사용자 승인 대기
"

### `retrospective` - 회고 스타일
"
**목적**: 학습과 개선을 위한 성찰적 접근
"
**WELL-LESS-Idea-Action 구조**:
```
# 🎯 프로젝트 회고: [프로젝트명]
"
## ✅ Well (잘된 점)
- **TDD 사이클 준수**: Alfred가 RED→GREEN→REFACTOR 철저히 관리
- **문서 동기화**: 코드 변경 즉시 반영으로 Living Document 유지
- **에이전트 협업**: spec-builder와 tdd-implementer의 시너지 효과
"
## 🔄 Less (아쉬운 점)
- **초반 요구사항 모호**: SPEC 재작성으로 2일 소요
- **성능 테스트 부족**: 나중에 병목 발견으로 리팩토링
- **Frontend-Backend 간 간극**: API 스펙 불일치 발생
"
## 💡 Idea (개선 아이디어)
- **Pre-Sprint 워크숍**: `/alfred:1-plan` 전에 요구사항 워크숍 추가
- **성능 게이트**: `/alfred:2-run`에 자동 성능 테스트 추가
- **API 계약 테스트**: Frontend용 전문 에이전트 활용
"
## 🎯 Action (행동 항목)
- [ ] 다음 스프린트부터 사전 요구사항 워크숍 의무화
- [ ] moai-domain-performance Skill 추가 개발
- [ ] API Contract Testing 프로세스 정립
"
## 🔑 학습 내용
1. **요구사 명확성**이 전체 개발 시간의 50%를 좌우
2. **자동화된 품질 게이트**가 수작업 검토보다 3배 효율적
3. **에이전트 전문성**이 복잡한 문제 해결의 핵심
"

### `tutorial` - 튜토리얼 스타일
"
**목적**: 단계별 학습과 실습 지원
"
**특징**:
- 친절하고 격려하는 톤
- 단계별 명확한 지침
- 예상 결과와 트러블슈팅
- 관련 학습 자료 링크
"
```
# 🚀 [튜토리얼 제목]
"
이 튜토리얼에서는 [학습 목표]를 단계별로 진행합니다. 약 15분 소요됩니다.
"
## 준비 사항
- Python 3.13+
- 터미널 접근 권한
- 기본 Git 지식
"
## Step 1: [첫 번째 단계]
먼저 프로젝트를 초기화해 보겠습니다.
"
```bash
mkdir my-project
cd my-project
moai-adk init
```
"
✅ **예상 결과**: `.moai/`와 `.claude/` 디렉터리가 생성됩니다.
"
❌ **문제 해결**: 권한 오류가 발생하면 `sudo` 없이 실행해 보세요.
"
## Step 2: [두 번째 단계]
이제 Alfred를 실행해 보겠습니다.
"
```bash
claude
/alfred:0-project
```
"
Alfred가 몇 가지 질문을 할 겁니다. 차근차근 답변해 주세요!
"
## 💡 팁
- 명령어가 기억나지 않으면 `/alfred:?`를 입력하세요
- 언제든 `Ctrl+C`로 중단하고 다시 시작할 수 있습니다
"
## 🔗 다음 단계
튜토리얼을 마쳤다면 [Alfred 가이드](./index)에서 고급 기능을 배워보세요!
```
