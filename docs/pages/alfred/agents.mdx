"
Alfred가 호출하는 서브 에이전트들은 각자 전문 분야를 가지며, 명확한 역할 분담을 통해 복잡한 개발 작업을 효율적으로 처리합니다.
"
## 에이전트 호출 원칙
"
### 계층 구조 준수
모든 에이전트는 다음 우선순위를 따릅니다:
1. **명령어 지침** - alfred:* 명령의 런북
2. **에이전트 정의** - `.claude/agents/alfred/*.md`의 역할 정의
3. **전문 분야** - 에이전트의 전문성 영역
"
### 자동 호출 조건
- **명시적 호출**: Alfred가 직접 특정 에이전트 지정
- **키워드 트리거**: SPEC/요구사항의 키워드 감지
- **상황적 호출**: 오류, 품질 검증, 특수 상황 발생
"
## 코어 에이전트 (10명)
"
### `spec-builder` - SPEC 작성 전문가
"
**역할**: 요구사항을 EARS 기반 SPEC으로 변환
"
**호출 조건**:
- `/alfred:1-plan` 실행 시
- 요구사항이 모호하거나 정제가 필요한 경우
"
**주요 책임**:
- EARS 5가지 패턴 적용
- 사용자 수준(초보/중급/전문가)에 맞는 설명
- 요구사항 모호성 탐지 및 질문 생성
"
**Skills**: `moai-foundation-specs`, `moai-alfred-ears-authoring`
"
### `implementation-planner` - 구현 전략가
"
**역할**: SPEC 분석과 구현 계획 수립
"
**호출 조건**:
- `/alfred:2-run` Phase 1
- 복잡한 기능의 구현 전략 필요 시
"
**주요 책임**:
- SPEC 해석과 작업 분해
- 테스트 시나리오 설계
- 위험 요소 분석과 회피 전략
- 기술 스택 결정
"
### `tdd-implementer` - TDD 실천가
"
**역할**: RED → GREEN → REFACTOR 사이클 실행
"
**호출 조건**:
- `/alfred:2-run` Phase 2
- TDD 워크플로우 필요 시
"
**주요 책임**:
- 실패하는 테스트 먼저 작성 (RED)
- 최소한의 코드로 통과 (GREEN)
- 코드 정리와 리팩토링 (REFACTOR)
- 적절한 언어별 Skills 활용
"
### `quality-gate` - 품질 관리자
"
**역할**: 코드 품질, 테스트, 보안 검증
"
**호출 조건**:
- `/alfred:2-run` Phase 2.5
- `/alfred:3-sync` 품질 확인
- PR 생성 전 검증
"
**주요 책임**:
- TRUST 5 원칙 검증
- 테스트 커버리지 확인 (85%+)
- 코드 리뷰 및 린트 검사
- 보안 취약점 스캔
"
### `doc-syncer` - 문서 동기화 전문가
"
**역할**: 코드 변경을 문서로 반영
"
**호출 조건**:
- `/alfred:3-sync` 실행 시
- 문서 업데이트 필요 시
"
**주요 책임**:
- README, CHANGELOG 자동 업데이트
- API 문서 최신화
- 보고서 생성 및 통계 정리
- Living Document 관리
"
### `tag-agent` - TAG 무결성 관리자
"
**역할**: @TAG 체인 검증과 고아 태그 탐지
"
**호출 조건**:
- `/alfred:3-sync` TAG 검증 시
- @TAG 무결성 의심 시
"
**주요 책임**:
- `@SPEC ↔ @TEST ↔ @CODE ↔ @DOC` 체인 연결 확인
- 고아 TAG 탐지 및 보고
- TAG 네이밍 규칙 검증
- 일관성 체크
"
### `git-manager` - Git 워크플로우 관리자
"
**역할**: Git 작업 자동화 및 브랜치 관리
"
**호출 조건**: 모든 단계에서 공통
"
**주요 책임**:
- 브랜치 생성 및 전환
- 커밋 메시지 작성 (Alfred 서명 포함)
- Pull Request 생성 및 관리
- 체크포인트 생성 및 복원
"
### `project-manager` - 프로젝트 관리자
"
**역할**: 프로젝트 구조와 메타데이터 관리
"
**호출 조건**:
- `/alfred:0-project` 실행 시
- 프로젝트 설정 변경 시
"
**주요 책임**:
- `.moai/` 디렉터리 구조 생성
- `.moai/config.json` 설정 관리
- 템플릿 버전 기록
- 프로젝트 메타데이터 유지
"
### `debug-helper` - 디버깅 전문가
"
**역할**: 오류 분석 및 해결 전략 제시
"
**호출 조건**:
- 런타임 오류 발생 시
- 빌드 실패 등 예기치 않은 상황
"
**주요 책임**:
- 오류 로그 분석
- 원인 파악 및 재현 시나리오 제시
- Fix 전략 제안
- 예방 조치 안내
"
### `trust-checker` - TRUST 검증자
"
**역할**: TRUST 5 원칙 준수 확인
"
**호출 조건**:
- 품질 검증 필요 시
- 코드 리뷰 단계
"
**TRUST 5 원칙**:
- **Test First**: 테스트 우선 작성
- **Readable**: 가독성 있는 코드
- **Unified**: 일관된 스타일
- **Secured**: 보안 규칙 준수
- **Trackable**: 변경 추적 가능
"
## 전문가 에이전트 (6명)
"
### `backend-expert` - 백엔드 전문가
"
**전문 분야**: API, 서버, 데이터베이스, 인증
**트리거 키워드**: backend, api, server, database, authentication
"
### `frontend-expert` - 프론트엔드 전문가
"
**전문 분야**: UI 컴포넌트, 상태 관리, 클라이언트 아키텍처
**트리거 키워드**: frontend, ui, component, client-side, web
"
### `ui-ux-expert` - UI/UX 전문가
"
**전문 분야**: 디자인 시스템, 접근성, UX 전략, 프로토타이핑
**트리거 키워드**: design, ux, accessibility, prototype, figma
"
### `devops-expert` - DevOps 전문가
"
**전문 분야**: CI/CD, 인프라, 컨테이너, 배포, 모니터링
**트리거 키워드**: deployment, docker, kubernetes, pipeline, infrastructure
"
