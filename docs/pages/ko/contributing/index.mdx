# 기여 가이드

MoAI-ADK 프로젝트에 기여해주셔서 감사합니다! 이 가이드는 프로젝트에 효과적으로 기여하는 방법을 안내합니다.

## 기여 방법

### 1. 이슈 생성

버그 리포트나 기능 제안:
- [GitHub Issues](https://github.com/yourusername/MoAI-ADK/issues)에서 이슈 생성
- 명확한 제목과 상세한 설명 제공
- 재현 단계 포함 (버그의 경우)

### 2. Pull Request 제출

코드 기여 프로세스:

```bash
# 1. Fork 및 Clone
git clone https://github.com/your-username/MoAI-ADK.git
cd MoAI-ADK

# 2. 개발 환경 설정
uv venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate
uv pip install -e ".[dev]"

# 3. Feature 브랜치 생성
git checkout -b feature/your-feature-name

# 4. 변경사항 작성 및 테스트
pytest
black src tests
mypy src

# 5. 커밋 및 푸시
git add .
git commit -m "feat: Add your feature description"
git push origin feature/your-feature-name

# 6. PR 생성
gh pr create --base develop
```

## 코드 스타일

### Python 코드

- **Formatter**: Black (line length: 100)
- **Linter**: Ruff
- **Type Checker**: mypy
- **Docstring**: Google style

```python
def example_function(param1: str, param2: int) -> bool:
    """Function description.

    Args:
        param1: Description of param1
        param2: Description of param2

    Returns:
        Description of return value

    Raises:
        ValueError: When invalid input provided
    """
    pass
```

### 커밋 메시지

Conventional Commits 형식 사용:

```
<type>(<scope>): <subject>

<body>

<footer>
```

**Types**:
- `feat`: 새로운 기능
- `fix`: 버그 수정
- `docs`: 문서 변경
- `style`: 코드 스타일 (formatting)
- `refactor`: 리팩토링
- `test`: 테스트 추가/수정
- `chore`: 빌드/설정 변경

**예시**:
```
feat(agents): Add new validation agent

Implement validation agent for TRUST 5 compliance checking.
Includes unit tests and documentation.

Closes #123
```

## 테스트 가이드라인

### 테스트 작성

모든 새로운 기능은 테스트를 포함해야 합니다:

```python
import pytest
from moai_adk import YourModule


def test_your_feature():
    """Test description."""
    # Arrange
    input_data = "test"

    # Act
    result = YourModule.process(input_data)

    # Assert
    assert result == expected_output


@pytest.mark.parametrize("input,expected", [
    ("test1", "result1"),
    ("test2", "result2"),
])
def test_multiple_cases(input, expected):
    """Parametrized test."""
    assert YourModule.process(input) == expected
```

### 테스트 실행

```bash
# 전체 테스트
pytest

# 특정 파일
pytest tests/test_your_module.py

# 커버리지 확인
pytest --cov=src --cov-report=html
```

### 테스트 커버리지

- 최소 80% 커버리지 유지
- 핵심 기능은 90% 이상 목표
- Edge case 테스트 포함

## 문서 작성

### Docstring

모든 public 함수/클래스에 docstring 작성:

```python
class ExampleClass:
    """Class description.

    This class demonstrates proper documentation.

    Attributes:
        attribute1: Description of attribute1
        attribute2: Description of attribute2

    Example:
        >>> obj = ExampleClass()
        >>> obj.method()
        'result'
    """

    def method(self, param: str) -> str:
        """Method description.

        Args:
            param: Parameter description

        Returns:
            Return value description
        """
        return param
```

### 마크다운 문서

docs/ 디렉토리의 문서 업데이트:

- 명확하고 간결한 설명
- 코드 예제 포함
- 스크린샷/다이어그램 (필요시)
- 내부 링크 활용

## 개발 워크플로우

### 로컬 개발

```bash
# 개발 서버 실행 (docs)
cd docs
npm run dev

# 패키지 개발
cd ..
uv pip install -e ".[dev]"

# 변경사항 테스트
pytest -v
```

### 코드 품질 체크

```bash
# Formatting
black src tests
isort src tests

# Linting
ruff check src tests

# Type checking
mypy src

# All checks
pre-commit run --all-files
```

### Pre-commit Hooks

자동 코드 품질 체크:

```bash
# Pre-commit 설치
pip install pre-commit
pre-commit install

# 수동 실행
pre-commit run --all-files
```

## Skills 개발
